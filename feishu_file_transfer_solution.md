# 飞书文件传输解决方案

> 问题：当前无法通过飞书直接发送文件
> 目标：实现从 OpenClaw 通过飞书发送文件给用户

---

## 问题诊断

### 当前权限状态
已获取权限：
- `im:message:send_as_bot` - 发送消息 ✅
- `im:message` - 消息相关 ✅
- `im:resource` - 资源访问（可能仅支持图片）⚠️

**缺失权限**：
- `im:file:write` - 上传文件 ❌
- `im:file:read` - 读取文件 ❌

---

## 方案一：申请飞书权限（推荐）

### 步骤1：进入飞书开发者后台
1. 访问 https://open.feishu.cn/app
2. 选择你的应用（OpenClaw 使用的应用）
3. 进入「权限管理」

### 步骤2：添加权限
搜索并添加以下权限：

| 权限名称 | 权限代码 | 用途 |
|---------|---------|------|
| 上传文件 | `im:file:write` | 上传文件到飞书 |
| 读取文件 | `im:file:read` | 读取文件信息 |
| 发送文件消息 | `im:message:send` | 发送文件类型消息 |

### 步骤3：发布版本
添加权限后需要：
1. 进入「版本管理与发布」
2. 创建新版本
3. 提交审核（或自用直接发布）

### 步骤4：验证
权限生效后，OpenClaw 将能直接发送文件。

---

## 方案二：使用 Kimi 中转（备选）

如果飞书权限申请复杂，可以使用当前方式：
- 文件通过 `kimi_upload_file` 发送到 Kimi 对话
- 用户在 Kimi 中下载

**优点**：无需额外配置，当前可用
**缺点**：需要切换到 Kimi 查看文件

---

## 方案三：Base64 编码发送（技术方案）

对于小文件（<1MB），可以将文件转为 Base64 通过消息发送：

```python
# 伪代码示例
import base64

with open("file.zip", "rb") as f:
    encoded = base64.b64encode(f.read()).decode()
    
# 发送 Base64 字符串
send_message(f"FILE:{filename}:{encoded}")
```

**限制**：
- 飞书单条消息有长度限制（约 20KB 文本）
- 只适合极小的文件

---

## 方案四：云存储中转

使用云存储作为中转：
1. 文件上传到云存储（如阿里云OSS、腾讯云COS）
2. 获取临时访问链接
3. 通过飞书发送链接

**需要配置**：
- 云存储账号和 API Key
- 在 OpenClaw 中添加云存储配置

---

## 建议操作顺序

1. **立即**：使用方案二（Kimi中转），当前可用
2. **今天**：申请飞书权限（方案一），一劳永逸
3. **后续**：如需频繁传大文件，考虑方案四（云存储）

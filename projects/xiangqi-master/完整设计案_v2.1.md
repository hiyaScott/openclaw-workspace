## 一、产品概述

### 1.1 产品定位
一款面向零基础用户的中国象棋教学App，借鉴多邻国(Duolingo)的游戏化学习模式，通过碎片化、关卡化、社交化的设计，让用户在轻松有趣的氛围中掌握中国象棋。

**核心理念**: "棋的核心是下" —— 所有教学通过实战下棋完成，而非问答模式

### 1.2 核心用户画像
| 用户类型 | 特征 | 需求 |
|---------|------|------|
| 象棋小白 | 完全不会，想入门 | 简单易懂，循序渐进 |
| 家长用户 | 想教孩子下棋 | 适合儿童，有成就感 |
| 业余爱好者 | 会一点，想系统学 | 查漏补缺，提升棋力 |
| 老年用户 | 休闲娱乐 | 大字模式，简单操作 |

### 1.3 竞品分析
| 产品 | 优点 | 缺点 | 我们的差异化 |
|-----|------|------|-------------|
| 多邻国国际象棋 | 游戏化设计优秀 | 没有中国象棋 | 本土化内容 + Max角色 |
| 象棋巫师 | 功能全面 | UI老旧，学习曲线陡 | 现代化UI，循序渐进 |
| 天天象棋 | 对战功能强 | 教学薄弱 | 专注教学，降低门槛 |

## 四、核心体验框架：章-主题-关-局

### 4.1 四层架构概述

象棋大师采用**四层递进式学习架构**，从宏观到微观，让用户在清晰的学习路径中逐步掌握象棋：

```
📚 章 (Chapter)          →  学习阶段，约10-20关
   └── 🎯 主题 (Topic)   →  知识模块，约8-12局
       └── ⭐ 关 (Level)  →  关卡节点，包含多个局
           └── 🎮 局 (Ju) →  最小学习单元，单局对战
```

**设计原则**：
- **章**：提供学习地图，让用户看到整体进度
- **主题**：聚焦单一知识点，避免信息过载
- **关**：游戏化节点，提供成就感和休息点
- **局**：实战练习，"棋的核心是下"

**设计逻辑**：
- **章**：提供长期目标感，每章约30-45分钟学习内容
- **主题**：聚焦单一知识点，降低认知负担
- **关**：可视化进度节点，形成闯关动力
- **局**：即时反馈单元，保持心流状态

### 4.2 章 (Chapter) - 学习阶段

**定义**：最大的学习单元，对应一个完整的知识模块

**数量**：12章，共150关

| 章 | 名称 | 主题数 | 预计时长 | 学习目标 |
|---|------|--------|---------|---------|
| 1 | 初识棋盘 | 6 | 30分钟 | 认识棋盘、棋子 |
| 2 | 走子基础 | 6 | 45分钟 | 掌握7种棋子走法 |
| 3 | 将帅安全 | 4 | 30分钟 | 理解将军、应将 |
| 4 | 基本战术 | 5 | 60分钟 | 捉双、牵制等 |
| 5 | 基本杀法 | 5 | 60分钟 | 对面笑、双车错等 |
| 6 | 开局基础 | 4 | 45分钟 | 常见开局 |
| 7 | 中局技巧 | 5 | 60分钟 | 攻守转换 |
| 8 | 残局基础 | 4 | 45分钟 | 单兵、单车残局 |
| 9 | 经典残局 | 3 | 30分钟 | 海底捞月等 |
| 10 | 名局赏析 | 2 | 20分钟 | 古谱学习 |
| 11 | 实战对弈 | 4 | 45分钟 | 综合应用 |
| 12 | 巅峰挑战 | 6 | 90分钟 | 高难度挑战 |

**界面呈现**：
- 卡片式列表，显示进度条
- 已完成：绿色勾选 + 奖杯
- 进行中：蓝色高亮 + Max提示
- 未解锁：灰色锁定

**数量**：每章4-6个主题，共54个主题

### 4.5 局 (Ju) - 单局练习

**定义**：最小的学习单元，一局完整的下棋练习

**数量**：每关8-12局，共约1200局

**单局时长**：30秒-2分钟

**局类型与提示分级**：
| 局类型 | 标签 | 提示方式 | 说明 |
|--------|------|---------|------|
| 教学局 | 📚 | 精确提示（虚线绿圈） | 新手引导 |
| 练习局 | ✏️ | 文字提示 | 巩固练习 |
| 巩固局 | 💪 | 模糊提示 | 独立思考 |
| 考试局 | 📝 | 无提示 | 能力检验 |

**提示分级系统**：
```
第1-3局（教学）：精确提示
├── 虚线绿圈标记目标位置
└── Max详细讲解操作步骤

第4-6局（练习）：文字提示
├── 如"找到可以吃掉对方兵的走法"
└── 不显示具体目标位置

第7-9局（巩固）：模糊提示
├── 如"找到最佳的下法，后面就看你的了……"
└── 完全依赖用户判断

第10局（考试）：无提示
└── 独立完成，检验掌握程度
```

**单局流程**：
```
1. 进入界面
   ├── 顶部：进度条（第X局/共Y局）
   ├── 中部：Max提示语
   └── 主体：棋盘

2. 用户操作
   ├── 选择棋子 → 音效+选中效果
   ├── 移动/吃子 → 音效+动画
   └── 错误 → 抖动+提示

3. 完成反馈
   ├── 成功：底部绿色横幅弹出
   │   ├── "泰裤辣！" + 对勾
   │   ├── 重玩/分享按钮
   │   └── 「继续」大按钮
   └── 自动进入下一局
```

## 五、功能实现详情（DEMO v7）

### 5.1 音效系统

**技术方案**: Web Audio API 合成音效，无需外部音频文件

**音效类型**:
| 音效 | 触发时机 | 技术实现 |
|-----|---------|---------|
| 选择 | 点击棋子 | 800Hz→400Hz 频率衰减 |
| 移动 | 棋子移动 | 600Hz→300Hz 正弦波 |
| 吃子 | 吃掉对方棋子 | 400Hz→200Hz 锯齿波 |
| 成功 | 完成一局 | C5-E5-G5-C6 四音上行 |
| 错误 | 错误移动 | 200Hz 低频提示 |

**控制**: 右上角 🔊/🔇 按钮切换

### 5.2 提示分级系统

**四级提示，逐步放开**:

| 级别 | 适用局 | 视觉标记 | 说明 |
|-----|-------|---------|------|
| 精确提示 | 教学局(1-3) | 虚线绿圈 | 最直观的目标指示 |
| 文字提示 | 练习局(4-6) | 无标记，文字引导 | "吃掉最近的棋子" |
| 模糊提示 | 巩固局(7-9) | 淡黄色圆点 | "找到最佳的下法" |
| 无提示 | 考试局(10+) | 无任何标记 | 完全独立判断 |

**实现逻辑**:
```javascript
hintType: 'precise' | 'text' | 'vague' | 'none'
showTargetCircle = hintType === 'precise' || (hintType === 'text' && currentStepIndex === 0)
```

### 5.3 多步关卡系统

**支持复杂连续操作**:
- 多步操作，每步可配置多个正确答案
- 步骤间自动推进，Max 给出阶段反馈

**配置示例**（第9关）:
```javascript
steps: [
  { target: [{x:6,y:6}, {x:3,y:5}], msg: '很好！现在可以吃掉对方的兵了！' },
  { target: {x:5,y:4, type:'capture'}, msg: '完美！移动和吃子都掌握了！' }
]
```

### 5.4 错误提示系统

**动态错误反馈**:
- 通用提示："不对哦，再想想看？"
- 精确提示关卡："不对哦，点击虚线绿圈的位置！"
- 文字提示关卡：结合 hintText 给出具体引导

**视觉反馈**: 棋盘抖动 + Max 提示语更新

### 5.5 局结算界面

**多邻国风格设计**:

- 底部：全宽「继续」按钮（带按压效果）

### 5.6 Max 形象应用

**小头像**（游戏界面）:

### 5.7 棋子渲染优化

**z-index 层级**:
```
z-index: 5   - 移动指示器（绿点）
z-index: 10  - 棋子（红马、黑子）
z-index: 20  - 目标区域（可点击）
z-index: 100 - 当前选中的马（最上层）
```

**渲染顺序**:
1. 先渲染其他棋子（被吃的子在底层）

## 六、关卡类型设计（7种核心类型）

### 4.2 单关结构（多邻国模式）

**每个大关包含 8-12 个小关卡**，小关卡连续进行，主题相同但难度递增。

**示例：第二章"认识马"**
- 小关卡 1-2：基础走法（四个方向）
- 小关卡 3-4：吃子练习
- 小关卡 5-6：逃脱危险
- 小关卡 7-8：蹩马腿教学
- 小关卡 9-10：综合应用

### 4.3 关卡类型设计（7种核心类型）

为保持关卡体验丰富，设计以下7种关卡类型：

#### 类型1：基础移动
**目标**：学习棋子基本走法
**特点**：
- 单一目标点
- 无威胁棋子
- 引导熟悉走法规则

**示例**：
```
马在 (4,7) → 移动到 (2,6)
提示："马走日字，试试看！"
```

#### 类型2：吃子练习
**目标**：学会用棋子吃子
**特点**：
- 目标位置有对方棋子
- 吃掉后对方棋子消失
- 正向反馈强化

**示例**：
```
马在 (4,7) → 吃掉 (2,6) 的炮
提示："用马吃掉对方的炮！"
```

#### 类型3：躲避危险
**目标**：从威胁下逃脱
**特点**：
- 己方棋子被对方棋子威胁
- 必须移动到安全位置
- 培养危机意识

**示例**：
```
马被车盯上了！
马在 (4,5) → 逃到 (2,4) 或 (6,4)
提示："快用日字走法逃脱！"
```

#### 类型4：吃子+躲避（综合）
**目标**：在复杂局面下决策
**特点**：
- 既有吃子机会也有危险
- 需要判断最优解
- 培养局势分析能力

**示例**：
```
马可以吃掉对方的卒，但会被炮反击
选择：吃子 vs 躲避
提示："小心！吃子后会被反击哦！"
```

#### 类型5：多方向选择（部分危险）
**目标**：识别安全走法
**特点**：
- 多个合法移动点
- 部分方向有危险
- 需要预判后果

**示例**：
```
马可以走到A、B、C三个位置
A：安全
B：会被车吃
C：安全
提示："哪个方向是安全的？"
```

#### 类型6：无提示挑战
**目标**：独立思考和判断
**特点**：
- 不显示合法移动提示
- 用户需自己计算走法
- 用于检验掌握程度

**示例**：
```
马在 (4,7)，目标吃掉 (3,5) 的兵
不显示绿色提示点
提示："自己算算马能走到哪？"
```

#### 类型7：连续博弈（多步对战）
**目标**：模拟真实对弈场景
**特点**：
- 双方轮流走棋
- 多步连续操作
- 包含吃子、躲避、选择的复杂组合

**示例**：
```
回合1：马吃卒
对方：车捉马
回合2：马躲避
对方：炮打马
回合3：马再次躲避并反击
提示："连续躲闪，寻找反击机会！"
```

### 4.4 关卡难度曲线

**难度递增规律**：
```
第1-2关：类型1（基础移动）
第3-4关：类型2（吃子练习）
第5-6关：类型3（躲避危险）
第7关：类型4（综合决策）
第8关：类型5（多方向选择）
第9关：类型6（无提示挑战）
第10关：类型7（连续博弈）
```

## 七、游戏化机制（继承多邻国5大特点）

### 7.1 特点1：单路径闯关

**设计理念**：2022年多邻国升级后，从自我规划变为单路径，用户只需按既定路线走，降低决策成本。

**我们的实现**：
```
用户路径：
注册 → 能力测试 → 第一章 → 主题1 → 第1关 → 第1局
                              ↓
                         第1关 → 第2局 → ... → 第10局
                              ↓
                         主题1完成 → 主题2 → ...
                              ↓
                         第一章完成 → 第二章 → ...
```

**界面体现**：
- 章界面：单列表，从上到下
- 主题界面：网格，已完成的在上面
- 关界面：单路径，节点沿路线分布
- 局界面：线性进度条，自动下一局

### 7.2 特点2：多样化关卡类型

**设计理念**：多种题型组合，保持学习新鲜感，避免单一枯燥。

**我们的关卡类型**（7种）：

| 类型 | 说明 | 占比 | 适用局 |
|------|------|------|--------|
| 基础移动 | 学习基本走法 | 30% | 教学局(1-3) |
| 吃子练习 | 学会吃子 | 20% | 教学局(1-3) |
| 躲避危险 | 识别威胁并逃脱 | 20% | 练习局(4-6) |
| 综合决策 | 吃子vs躲避权衡 | 15% | 练习局(4-6) |
| 多方向选择 | 识别安全路线 | 10% | 巩固局(7-9) |
| 无提示挑战 | 独立计算 | 3% | 巩固局(7-9) |
| 连续博弈 | 多回合对战 | 2% | 考试局(10) |

### 6.3 特点3：80%正确率难度

**设计理念**：既让用户有挑战，又不至于太难放弃。多邻国通过AI动态调整，保持80%正确率。

**我们的难度曲线**：
```
关卡1-3:  简单（⭐）    正确率目标: 90%+
关卡4-6:  中等（⭐⭐）  正确率目标: 80%+
关卡7-9:  困难（⭐⭐⭐）正确率目标: 70%+
关卡10:   挑战（⭐⭐⭐）正确率目标: 60%+
```

**难度调节机制**：
- 连续错误2次 → Max主动提示
- 连续正确3次 → 减少提示，增加挑战
- 关卡10允许失败重试，不惩罚

### 6.4 特点4：即时反馈系统

**设计理念**：每答对一题都有音效和动画反馈，让用户立即感受到成就感。

**我们的反馈设计**：

| 操作 | 视觉反馈 | 音效反馈 | 触觉反馈 |
|------|---------|---------|---------|
| 选择棋子 | 脉冲光环 | 清脆音阶 | 轻微震动 |
| 移动棋子 | 涟漪扩散 | 柔和sine波 | - |
| 正确 | 星星飞散 | 胜利旋律 | 成功震动 |
| 错误 | 棋盘抖动 | 低沉提示 | 错误震动 |
| 关卡完成 | 彩带飘落 | 庆祝音效 | 长震动 |

**反馈时机**：
- 即时：操作后100ms内反馈
- 清晰：正确绿色，错误红色
- 愉悦：动画流畅，音效和谐

### 6.5 特点5：连胜机制（Streak）

**设计理念**：连续登录奖励，形成学习习惯。多邻国的连胜机制是用户留存的核心。

**我们的连胜设计**：

**连胜显示**：
- 顶部固定显示 🔥 连胜X天
- 火焰动画吸引注意
- 连续天数越多，火焰越大

**连胜奖励**：
| 连胜天数 | 奖励 |
|---------|------|
| 3天 | 10宝石 |
| 7天 | 30宝石 + 专属徽章 |
| 14天 | 50宝石 + 双倍经验卡 |
| 30天 | 100宝石 + 限定头像框 |

**连胜保护**：
- 每周可使用1次"连胜冻结"
- 错过1天不会断连胜
- 冻结道具通过完成关卡获得

**连胜提醒**：
- 每日推送提醒学习
- 连续3天未学习，Max主动关心
- 断连胜后，鼓励重新开始

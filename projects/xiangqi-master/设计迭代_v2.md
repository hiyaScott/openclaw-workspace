## 一、核心设计原则

### 1.1 学习理念
**"棋的核心是下"** —— 所有教学通过实战下棋完成，而非问答模式

### 1.2 关卡结构
```
📚 章（阶段）→ 关（关卡节点）

示例：
第一章：初识棋盘 (10关)
├── 第1关：认识九宫（实战下棋）
├── 第2关：认识楚河汉界（实战下棋）
└── ...
```

## 二、美术风格规范

### 2.1 主题角色：Max

**角色定位**: 象棋教练 / 学习伙伴 / AI指引者

**角色设定**: Max 即是我（Kimi Claw）——玩家的AI象棋教练，以蓝色极简形象陪伴学习每一步

**设计特征**:
| 元素 | 规范 |
|------|------|
| 头部 | 白色圆形，极简两点眼睛 |
| 服装 | 蓝色斗篷（主色 #2196F3）|
| 线条 | 粗黑边线（3-4px）|
| 风格 | 矢量扁平、圆角友好 |

**角色应用**:
- 关卡路径中陪伴用户
- 对话框中提供提示和鼓励
- 成就/升级时庆祝动画

### 2.2 色彩系统

**主色调**:
| 用途 | 色值 | 说明 |
|------|------|------|
| Max蓝 | #2196F3 | 品牌主色，角色服装 |
| 成功绿 | #4CAF50 | 完成状态、正确反馈 |
| 警告黄 | #FFC107 | 提示、待完成 |
| 错误红 | #F44336 | 错误反馈 |
| 背景灰 | #F5F5F5 | 页面背景 |
| 棋盘木 | #D4A574 | 棋盘底色 |

### 2.3 棋盘设计

**尺寸规范**:
- 棋盘占画面 **70-80%** 宽度
- 中国象棋：9×10 网格
- 格子大小：适配手指触控（最小 44px）

**棋子设计**:
- 矢量扁平风格
- 红方：#D32F2F（中国红）
- 黑方：#424242（墨黑）
- 字体：楷体 / 思源宋体

## 四、关卡内容重构

### 4.1 第一章：初识棋盘（原问答 → 改为实战）

| 原设计 | 新设计 |
|--------|--------|
| 选择题：九宫是什么？ | 实战：将帅走到九宫中心 |
| 选择题：楚河汉界作用？ | 实战：移动棋子过河 |
| 选择题：将怎么走？ | 实战：将帅移动练习 |
| ... | ... |

**核心原则**: 每个关卡都是一次"下棋练习"，在实战中理解规则

### 4.2 教学模式

**引导式学习**:
1. Max 提示目标（"用马吃掉对方的炮"）
2. 用户尝试操作
3. 正确 → 庆祝动画 + 下一关
4. 错误 → Max 提示 + 允许重试

**提示系统**:
- 灯泡按钮显示提示（消耗宝石）
- Max 在困难时主动鼓励

## 五、交互规范

### 5.1 动画效果

| 场景 | 动画 |
|------|------|
| 棋子移动 | 平滑滑动 300ms |
| 吃子 | 缩放消失 + 粒子效果 |
| 关卡完成 | Max 庆祝动画 |
| 升级 | 全屏彩带 + 段位徽章 |

### 5.2 反馈设计

- **正确**: 绿色高亮 + 清脆音效
- **错误**: 轻微震动 + Max 鼓励
- **完成**: 星星收集动画

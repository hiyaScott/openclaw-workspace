# 发布哨兵 (Deploy Sentinel)

**一句话描述**: 标准化发布流程，拦截问题代码，确保部署成功率。

**触发方式**: 对我说 `/deploy-check` 或 `/发布检查`

---

## 能力范围

### ✅ 做的事情
- **构建检查**: 验证项目能否正常构建
- **链接检查**: 检查 HTML 中的死链、404
- **文件检查**: 验证关键文件存在性、大小合理性
- **Git 状态检查**: 确认分支、提交状态
- **推送执行**: 检查通过后自动推送到远程

### ❌ 不做的事情
- 不运行单元测试（项目无测试框架时跳过）
- 不检查性能指标
- 不做浏览器自动化测试

---

## 使用场景

| 场景 | 示例 |
|------|------|
| 发布前检查 | "我要发布 portfolio-blog，帮我检查一下" |
| 紧急修复后 | "刚改了首页链接，检查完直接推" |
| 批量更新后 | "更新了 10 个游戏页面，检查是否有死链" |

---

## 检查清单

```
□ 项目构建成功
□ HTML 无语法错误
□ 所有链接可访问
□ 关键文件存在
□ 文件大小合理
□ Git 工作区干净
□ 推送到远程成功
```

## 失败处理

如果检查失败，我会：
1. 列出具体问题
2. 给出修复建议
3. 询问是否继续发布

---

## 技术细节

- **适用项目**: 静态网站 (HTML/CSS/JS)
- **部署目标**: GitHub Pages
- **检查耗时**: 10-30 秒
- **自动修复**: 无（只报告，不修改代码）

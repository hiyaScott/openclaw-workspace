# 剧本写作 (Screenwriting)

## 概述

本技能涵盖影视剧本写作与游戏叙事设计的核心知识体系，包括传统三幕结构、角色塑造、对白技巧，以及游戏特有的分支叙事、环境叙事和关卡剧情设计。

---

## 一、影视剧本基础

### 1.1 剧本格式规范

#### 场景标题 (Scene Heading)
```
INT. 咖啡馆 - DAY
EXT. 山顶观景台 - NIGHT
```

**格式要素**:
- **场景性质**: `INT.`(内景) / `EXT.`(外景)
- **地点**: 具体位置名称
- **时间**: `DAY`(日) / `NIGHT`(夜) / `DAWN`(黎明) / `DUSK`(黄昏)

#### 动作描述 (Action/Scene Description)
- 用现在时态、主动语态
- 描述**可见的内容**: 动作、表情、环境、道具
- 描述**可听到的关键声音**
- 新角色首次出现时**名字全大写**，可附带年龄/外貌/特质
- 避免文学性抒情和心理描写

#### 对话格式
```
          李明 (30岁，疲惫的程序员)
     (揉着太阳穴)
      deadline 是明天，我连一半都没做完。
```

- **角色名**: 居中，全大写
- **表演指示**: 圆括号内，字体较小
- **对话内容**: 简洁、有潜台词

### 1.2 三幕结构 (Three-Act Structure)

| 幕 | 页数 | 占比 | 核心内容 |
|---|-----|------|---------|
| 第一幕 (建置) | p1-p30 | 25% | 介绍人物、世界、日常；p10-12设置背景；第一幕末出现**激励事件** |
| 第二幕 (对抗) | p30-p90 | 50% | 主角追求目标，遭遇挫折；p60为**中点**；第二幕末陷入**最低谷** |
| 第三幕 (解决) | p90-p120 | 25% | **高潮对决**；解决主要冲突；p110-112为高潮；余韵结尾 |

> **核心原则**: 所有故事都有开始、中间和结束。结构不是僵化的公式，而是组织故事、控制节奏的工具。

### 1.3 布莱克·斯奈德节拍表 (Save the Cat Beat Sheet)

| 节拍 | 页码 | 说明 |
|-----|------|------|
| 开场画面 | p1 | 展示主角的"之前"状态，基调、情绪、风格 |
| 阐明主题 | p5 | 有人提出问题/给出建议，主角不以为然 |
| 铺垫 | p1-p10 | 介绍人物、世界、缺陷、欲望 |
| 催化剂 | p12 | 打破日常的突发事件 |
| 争执 | p12-25 | 主角犹豫是否踏上旅程 |
| 进入第二幕 | p25 | 主角做出选择，世界转变 |
| B故事 | p30 | 爱情故事/友情/师徒关系的起点 |
| 游戏时间 | p30-55 | 主角探索新世界，看似轻松 |
| 中点 | p55 | 虚假的胜利或失败的假象，抬高赌注 |
| 反派逼近 | p55-75 | 反派重整旗鼓，内部矛盾显现 |
| 一无所有 | p75 | 失去一切，灵魂黑夜降临 |
| 进入第三幕 | p85 | 解决方案出现 |
| 最终对决 | p85-110 | 与反派决战 |
| 结局画面 | p110 | 展示主角的"之后"状态，与开场呼应 |

### 1.4 角色塑造

#### Want vs Need
- **Want (欲望)**: 角色表面上想要的东西（外在目标）
- **Need (需求)**: 角色内心深处真正需要的东西（内在成长）

#### 人物弧光 (Character Arc)
好的角色在故事中经历成长或转变：
- 认知上的变化
- 情感上的成熟
- 行为模式的改变

#### 反派设计
主角的品质取决于反派的品质。强大的反派才能衬托出主角的伟大。

## 二、游戏剧本写作

### 2.1 互动叙事核心

#### 玩家自主权 (Agency)
- 玩家需要被置于体验的中心
- 选择必须在关键时刻产生**实质影响**
- 避免"假选择"——看似有选项，实则无关紧要

#### 分支叙事结构

**珍珠串结构 (String of Pearls)**
```
主线A → [支线B] → 主线A → [支线C] → 主线A → 结局
```
- 线性主干 + 可选分支
- 分支不影响主线，但增加探索感

**分支与瓶颈结构 (Branch and Bottleneck)**
```
        → 路线B ─┐
起点 → 路线A ─┼→ 汇合点 → 结局
        → 路线C ─┘
```
- 玩家可以选择不同路径
- 但最终会汇合到主要叙事线

**真正的开放世界**
- 完全非线性的叙事结构
- 玩家自由探索，从不同角度接近故事节点
- 适合现实世界位置或探索型游戏

### 2.2 分支叙事设计原则

| 原则 | 说明 |
|-----|------|
| **有意义的选择** | 每个选择都应有可感知的后果 |
| **情绪投入** | 选择点应涉及情感或道德的权衡 |
| **信息透明** | 玩家应大致了解选择的可能方向（但不必完全透明） |
| **状态跟踪** | 使用变量系统跟踪玩家的选择历史 |
| **代价与收益** | 每个选择都有得有失，没有完美选项 |

### 2.3 叙事工具

**Ink** (Inkle Studios)
- 轻量级叙事脚本语言
- 与游戏引擎解耦
- 适合复杂对话树和分支逻辑

**Articy: Draft**
- 可视化叙事设计工具
- 流程图形式呈现分支结构
- 整合角色、地点、变量

**Twine**
- 开源、免费
- 快速原型工具
- 适合网页演示

**Arrow (Godot 4)**
- 开源可视化叙事工具
- 节点式设计
- 支持多语言和导出HTML

## 三、关卡/箱庭剧情设计

### 3.1 环境叙事 (Environmental Storytelling)

**定义**: 通过环境元素讲述故事，而非直接的文字或对话。

#### 环境叙事金字塔
```
        环境叙事 (顶层)
              ↑
        关卡设计 (中层)
              ↑
    世界架构/设定 (底层)
```

#### 环境叙事三要素

| 要素 | 说明 | 案例 |
|-----|------|------|
| **理解** | 环境暗示玩家到来前发生了什么 | 《生化奇兵》中的战斗痕迹 |
| **感受** | 建筑唤起特定情绪 | 《风之旅人》中的废墟之美 |
| **身份认知** | 环境提供角色背景信息 | 《艾迪芬奇的记忆》中各房间装饰 |

### 3.2 魂类箱庭设计精髓

#### 核心理念
1. **立体结构**: 在有限空间内构造自由感
2. **环形路径**: "一条直线能到的路硬是要绕一个圈"

#### 环形路径设计
```
    ┌────────────────┐
    │    起点        │
    │     ↓          │
    │   探索路径      │
    │     ↓          │
    │   解锁捷径 ←───┘
    └────────────────┘
         ↓
       篝火/检查点
```

**设计意图**:
- 中和死亡负反馈
- 制造"峰回路转"的顿悟时刻
- 增强空间认知的成就感

### 3.3 关卡环境分类

| 类型 | 特点 | 案例 |
|-----|------|------|
| **城镇** | 高密度、立体建筑、环形路线 | 魂系列的不死镇、史东薇尔城 |
| **地牢** | 狭窄、线性或分支、向下/向上 | 塞恩古城、地下墓地 |
| **平原** | 大面积、地形起伏、点缀遗迹 | 黑森林庭院、法兰要塞 |
| **教堂** | 单体建筑、楼层拆分、室内外穿插 | 幽邃教堂、净身小教会 |

### 3.4 关卡密度设计

**规律**: 关卡流程从复杂到精简，最终在Boss战前收束为线性

```
前期(复杂) → 中期 → 后期(线性) → Boss
  ↓              ↓          ↓
多点选择      路径收束     单一通道
分支支线      减少干扰     专注战斗
```

**人性化设计**: 在玩家兴致盎然时赋予复杂选择，在玩家疲惫时收束路径。

### 3.5 关卡叙事流程设计

**米那斯提利斯案例**:

| 阶段 | 区域 | 情绪 | 结构 |
|-----|------|------|------|
| 前期 | 城墙 → 中段城镇 | 积极、探索 | 线-面结构 |
| 中期 | 环山隧道 → 上部城镇 | 兴奋、期待 | 线-面结构 |
| 后期 | 顶峰广场 → 教堂 | 满足、休憩 | 面-体结构 |
| 终章 | 烽火台 → 试炼场 | 迷茫、深入 | 点-线结构 |

### 3.6 视觉引导技巧

| 技巧 | 说明 | 案例 |
|-----|------|------|
| **形状引导** | 三角形(引导)、矩形(遮挡)、圆形(聚集) | 平原上的石头、树木布局 |
| **光线引导** | 强光源、魔法绸缎、发光物体 | 《风之旅人》中的光柱 |
| **动态元素** | 炊烟、鸟群、流动的水 | 《看火人》中的视觉引导 |
| **可交互文字** | 3D悬浮文字指示路径 | 《艾迪芬奇的记忆》 |

### 3.7 关卡与叙事结合的设计流程

1. **前期调研**: 制作情绪板 (Moodboard)
2. **流程图**: 绘制宏观关卡流程
3. **节奏表**: 使用 Beatsheet 规划每个区域的情绪唤起
4. **标准键**: 确定关卡的标准尺寸和比例
5. **3D草模**: 搭建白盒 (Whitebox) 测试
6. **敌人分布**: 遵循"进化与增量"原则
7. **美术渲染**: 区域色彩、灯光、装饰

## 四、实用技巧

### 4.1 对白写作原则

1. **每句对白都有目的** - 推进剧情或揭示角色
2. **潜台词** - 角色说的 ≠ 角色想的
3. **简洁** - 剪掉废话，保留精华
4. **符合角色身份** - 语言、语调、节奏因人而异
5. **避免解释性对白** - "你知道，自从三年前那场车祸..."

### 4.2 视觉化写作

- 少用形容词和副词
- 多用具体的动作和可见的细节
- 通过行为、表情、环境互动展现心理

**错误示例**:
> 李明感到绝望，他觉得自己再也无法完成任务了。

**正确示例**:
> 李明盯着屏幕上的错误提示，手指悬在键盘上颤抖。他抓起咖啡杯，发现已经空了。

### 4.3 冲突设计

| 类型 | 说明 |
|-----|------|
| **人与他人** | 主角与反派、盟友之间的矛盾 |
| **人与环境** | 对抗自然、制度、时代背景 |
| **人与自我** | 欲望与道德、理智与情感的斗争 |

**黄金法则**: 没有冲突就没有戏剧。

# 战棋游戏设计师 (SRPG Designer)

## 能力定位

战棋游戏设计师是专注于**策略角色扮演游戏（SRPG/Tactical RPG）**的专业能力，综合了：

- **网络游戏设计师**的系统架构、数值平衡、经济系统设计能力
- **游戏设计**的玩法循环、心流体验、关卡设计能力
- **战棋品类专项**的网格战斗、策略维度、养成体系设计能力

## 核心能力维度

### 1. 战棋基础系统
- 网格/格子系统（正方形/六边形）
- 回合制战斗框架
- 移动与行动点系统
- 高低差与地形机制

### 2. 关卡设计（强化维度）
- 关卡三维框架：敌人-环境-机制
- 策略维度设计：敌人机制-地形机制-第三方机制
- 目标多样性：歼灭/守护/撤离/斩首
- 情感曲线与节奏控制

### 3. 战斗设计（强化维度）
- 职业/兵种设计与克制关系
- 技能与连携系统
- 武器三角/属性克制
- 掩体与视线系统（XCOM类）

### 4. 养成设计（强化维度）
- 角色成长曲线
- 职业转职系统
- 装备与武器强化
- 兵种搭配与携带系统

### 5. 数值与平衡
- 战棋战斗公式设计
- 移动力与射程平衡
- 养成与关卡难度匹配
- 长线运营数值规划

## 设计框架

### 战棋关卡设计三维模型

```
关卡基础框架:
                    额外机制
                         │
                         │
           敌人强度 ─────┼───── 环境
                         │
                        / \
                       /   \
                      /     \

坐标取点决定体验偏向:
- 偏向敌人 → 数值验证
- 偏向机制 → 策略验证
- 偏向环境 → 空间利用
```

### 策略维度模型

```
策略深度坐标系:

第三方机制 (油桶/机关/道具)
              │
              │    ● 高策略复杂度
              │   / 
              │  /   (学习成本高)
              │ /
敌人机制 ─────┼───── 地形机制
(技能/兵种)   │      (高低差/地形效果)
              │
              │
              ● 低策略复杂度
```

### 养成-关卡验证循环

```
养成即策略:

角色成长 → 新技能/属性 → 关卡策略变化 → 成长验证
    ↑                                          │
    └──────────── 数值/技巧成长反馈 ←──────────┘

关键原则:
- 不能只涨数值，策略要有变化
- 关卡要验证养成结果
- 不同养成路线 = 不同关卡策略
```

## 快速参考

### 经典战棋公式

#### 伤害公式
```
伤害 = (攻击力 - 防御力) × 克制系数 × 地形系数 × 随机波动

或百分比减伤:
伤害 = 攻击力 × (1 - 防御力/(防御力+常数))
```

#### 命中率公式
```
命中率 = 基础命中 + 技巧差值 - 地形闪避 - 距离惩罚

通常上限: 90-95%
通常下限: 5-10%
```

### 职业克制三角

```
    步兵/剑
     /     \
   /         \
 枪兵/长柄 ─── 骑兵/马

克制效果: 通常 1.2x - 1.5x 伤害
```

### 移动力基准

```
移动力参考:
- 步兵: 4-5格
- 骑兵: 6-8格 (通常只能直线移动)
- 飞行: 5-6格 (无视地形)
- 重甲: 3-4格

射程参考:
- 近战: 1格
- 长柄: 1-2格
- 弓箭: 2-3格
- 法术: 1-3格 (通常有最小射程限制)
```

## 最佳实践

1. **策略先于数值** - 先有有趣的策略选择，再平衡数值
2. **地形是第三位角色** - 好的地形设计创造策略空间
3. **养成必须改变策略** - 不只是数值成长，要有新选择
4. **控制单局时长** - 手游SRPG单关15-20分钟为宜
5. **保留悔棋空间** - 降低试错成本，鼓励策略尝试

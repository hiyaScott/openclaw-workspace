# 战棋基础系统 (SRPG Fundamentals)

## 战棋游戏定义

SRPG (Strategy Role-Playing Game) 是结合了策略游戏和角色扮演游戏元素的游戏类型，核心特征是**基于网格的回合制战斗**。

## 网格系统

### 网格类型

#### 正方形网格 (Square Grid)
```
┌───┬───┬───┬───┐
│   │   │   │   │
├───┼───┼───┼───┤
│   │   │   │   │
├───┼───┼───┼───┤
│   │   │   │   │
└───┴───┴───┴───┘

优点:
- 直观易懂
- 计算简单
- 地图编辑方便

缺点:
- 对角线移动距离问题 (√2 ≈ 1.414)
- 解决方案: 对角线消耗2移动力 或 禁止对角线移动
```

#### 六边形网格 (Hex Grid)
```
   / \   / \   / \
  /   \ /   \ /   \
 │     │     │     │
  \   / \   / \   /
   \ /   \ /   \ /
    │     │     │

优点:
- 各方向移动距离一致
- 更自然的地形表现
- 包围战术更流畅

缺点:
- 地图编辑较复杂
- 玩家学习成本略高
```

### 坐标系统

```python
# 正方形网格坐标
class SquareCoord:
    def __init__(self, x, y):
        self.x = x
        self.y = y
    
    def distance_to(self, other):
        # 曼哈顿距离 (4方向移动)
        return abs(self.x - other.x) + abs(self.y - other.y)
        
        # 切比雪夫距离 (8方向移动)
        # return max(abs(self.x - other.x), abs(self.y - other.y))

# 六边形网格坐标 (立方坐标系)
class HexCoord:
    def __init__(self, q, r, s):
        self.q = q  # 轴向x
        self.r = r  # 轴向z
        self.s = s  # 轴向y (q + r + s = 0)
    
    def distance_to(self, other):
        return (abs(self.q - other.q) + 
                abs(self.r - other.r) + 
                abs(self.s - other.s)) // 2
```

## 回合制框架

### 回合结构

```
玩家回合:
├─ 选择单位
├─ 移动阶段 (可选)
├─ 行动阶段 (攻击/技能/物品)
└─ 结束回合

敌方回合:
└─ AI执行所有单位行动

变体:
- 玩家阶段/敌方阶段 (所有单位行动完才切换)
- WT值系统 (按速度决定行动顺序)
- 即时制暂停 (战场女武神)
```

### WT值系统 (Weight Turn)

```
源自《皇家骑士团》的回合制变体:

每个单位有: 速度(SPD)
WT = 基础WT - SPD

行动顺序:
1. 所有单位按当前WT排序
2. WT最低的单位先行动
3. 行动后该单位WT重置为 (基础值 - SPD)
4. 其他单位WT递减
5. 重新排序，下一位行动

优点:
- 速度差异更直观
- 可以插入"准备"机制 (WT+等待)
- 策略性更强

缺点:
- 学习成本高
- 难以预测行动顺序
```

## 移动系统

### 移动范围计算

```python
# 基于BFS的移动范围计算
def calculate_movement_range(start, movement_points, grid):
    reachable = {start: 0}  # 位置: 消耗
    queue = [(start, 0)]
    
    while queue:
        current, cost = queue.pop(0)
        
        for neighbor in grid.get_neighbors(current):
            terrain_cost = grid.get_terrain_cost(neighbor)
            new_cost = cost + terrain_cost
            
            if new_cost <= movement_points:
                if neighbor not in reachable or new_cost < reachable[neighbor]:
                    reachable[neighbor] = new_cost
                    queue.append((neighbor, new_cost))
    
    return reachable
```

### 地形影响

```
地形类型与移动力消耗:

平地:   1.0x (基准)
森林:   1.5x (通常+闪避)
山地:   2.0x (通常+防御)
河流:   2.5x (骑兵不可通行)
墙壁:   不可通行

高低差影响:
- 向上移动: +0.5x 消耗每层高差
- 向下移动: 正常消耗
- 超过最大高低差: 不可通行
```

### Zone of Control (ZOC)

```
控制区域机制:

当敌人位于相邻格子时:
├─ 移动力消耗增加
├─ 无法直接穿过
└─ 离开ZOC可能需要额外消耗

用途:
- 防止直接穿越前线
- 强化阵地感
- 保护脆弱单位
```

## 战斗基础

### 基本战斗流程

```
1. 选择攻击者和目标
2. 检查射程和视线
3. 计算命中率
4. 命中判定 (随机数)
5. 计算伤害
6. 应用伤害和效果
7. 检查死亡/状态变化
```

### 命中率公式

```
基础公式:
命中率 = 命中方技巧 - 被命中方闪避 + 地形修正 + 距离修正 + 其他修正

示例:
攻击者技巧: 85
防御者闪避: 30 (基础20 + 森林地形10)
距离惩罚: -10
其他修正: +5 (侧面攻击)

命中率 = 85 - 30 - 10 + 5 = 50%

通常设置上限 (90-95%) 和下限 (5-10%)
```

### 伤害公式

```
减法公式 (日式SRPG常用):
伤害 = max(1, 攻击力 - 防御力) × 克制系数 × 其他系数

除法公式 (更平滑):
伤害 = 攻击力 × (攻击力 / (攻击力 + 防御力)) × 其他系数

示例 (减法):
攻击力: 50
防御力: 30
克制系数: 1.2 (兵种克制)
暴击: 1.5

伤害 = (50 - 30) × 1.2 × 1.5 = 36
```

### 连携/协力攻击

```
设计变体:

1. 位置连携
   多个友军包围敌人时触发
   每个相邻友军 +10% 伤害

2. 武器连携
   特定武器组合触发
   如: 剑+枪同时攻击同一目标

3. 技能连携
   A使用技能后，B对同一目标攻击有加成

4. 羁绊连携 (火纹)
   好感度高的角色相邻时
   概率触发协助攻击或防御
```

## 胜负条件

### 常见胜利条件

```
歼灭型:
├─ 击败所有敌人
├─ 击败敌方首领
└─ 击败指定数量敌人

目标型:
├─ 到达指定地点
├─ 保护友方单位存活X回合
├─ 护送单位到达目标点
└─ 坚守阵地X回合

收集型:
├─ 占领所有据点
├─ 收集指定物品
└─ 激活特定机关

混合型:
└─ 多种条件组合
```

### 可选任务与成就

```
主目标之外的挑战:

回合挑战:
├─ X回合内通关
└─ 超过X回合则评价降低

损失限制:
├─ 无阵亡通关
├─ 特定角色必须存活
└─ 损失不超过X个单位

特殊条件:
├─ 仅使用特定职业通关
├─ 不消耗道具通关
└─ 击败隐藏敌人

奖励:
└─ 额外资源/装备/角色
```

## 游戏模式

### 经典模式

```
《火焰纹章》模式:
├─ 线性剧情关卡
├─ 角色永久死亡 (可选)
├─ 有限资源管理
└─ 高策略密度

《梦幻模拟战》模式:
├─ 佣兵系统 (主将带兵)
├─ 多转职路线
├─ 经典关卡复刻
└─ 强调兵种克制
```

### 现代手游模式

```
养成导向:
├─ 每日挑战关卡
├─ 素材本/装备本
├─ 竞技场PVP
└─ 公会战/团战

Roguelike融合:
├─ 随机地图生成
├─ 局内成长选择
├─ 多周目解锁
└─ 高重玩价值
```

# 养成系统设计 (Progression Systems)

## 战棋游戏养成核心

### 养成即策略

```
核心原则:

角色成长 → 策略选择增加 → 关卡策略变化 → 成长验证

反例 (错误设计):
- 只涨数值，玩法不变
- 优势策略永远是优势策略
- 换汤不换药的重复体验

正例 (正确设计):
- 新技能带来新打法
- 不同养成路线 = 不同策略
- 关卡需要新策略验证
```

## 角色成长系统

### 等级系统

```
经验获取:
├─ 击败敌人: 基础经验
├─ 使用技能: 少量经验
├─ 完成目标: 额外奖励
└─ 生存奖励: 战后结算

升级效果:
├─ 属性成长 (随机/固定)
├─ 技能解锁
├─ 职业进阶资格
└─ 能力上限提升

《火焰纹章》成长模式:
- 每个角色有成长率
- 每次升级随机提升属性
- 凹点文化 (S/L追求最优)
- 个体差异带来独特角色
```

### 属性成长设计

```
成长曲线类型:

早熟型:
├─ 前期成长快
├─ 后期成长慢
└─ 适合: 降低前期难度

晚成型:
├─ 前期成长慢
├─ 后期成长快
└─ 适合: 培养成就感

平均型:
├─ 成长稳定
├─ 无突出优势
└─ 适合: 稳定可靠

随机型:
├─ 完全随机成长
├─ 充满不确定性
└─ 适合: 重玩价值

设计建议:
- 主要角色应有独特成长曲线
- 避免所有角色同质化
- 提供重置/优化机会 (手游)
```

## 职业转职系统

### 转职树设计

```
《梦幻模拟战》转职示例:

        战士 (初始)
       /          \
    剑士          枪兵
    /  \          /  \
  剑圣  英雄    骑士  龙骑
  
转职条件:
├─ 等级要求 (如Lv.10)
├─ 材料消耗 (转职道具)
└─ 前置职业 (必须先转职某职业)

跨系转职:
- 允许从剑士转枪兵
- 保留部分已学技能
- 提供极大自由度
```

### 职业平衡

```
职业差异化:

定位差异:
├─ 坦克: 高防低攻，保护队友
├─ 输出: 高攻低防，快速击杀
├─ 辅助: 低攻防，治疗增益
└─ 特殊: 独特机制，特定场合

数值差异:
├─ 移动力不同
├─ 射程不同
├─ 装备限制不同
└─ 技能类型不同

克制关系:
- 三角克制增加策略
- 无明显优势职业
- 每种职业都有上场机会
```

## 装备系统

### 装备类型

```
武器:
├─ 攻击属性加成
├─ 可能附带特效
├─ 职业限制
└─ 耐久度 (部分游戏)

防具:
├─ 防御属性加成
├─ 可能附带抗性
├─ 重量影响移动力
└─ 职业限制

饰品:
├─ 属性小幅度加成
├─ 特殊效果
└─ 无职业限制

套装效果:
├─ 2件套: 基础加成
├─ 4件套: 强力效果
└─ 鼓励收集完整套装
```

### 装备获取与强化

```
获取途径:
├─ 关卡掉落
├─ 商店购买
├─ 任务奖励
├─ 合成制作
└─ 抽卡获得 (手游)

强化系统:
├─ 升级: 提升基础属性
├─ 精炼: 解锁额外属性
├─ 附魔: 添加随机特效
└─ 突破: 提升等级上限

强化材料:
├─ 通用素材 (金币/矿石)
├─ 特定素材 (BOSS掉落)
└─ 同类型装备 (吞噬)
```

## 兵种/佣兵系统

```
《梦幻模拟战》佣兵系统:

机制概述:
├─ 英雄作为主将
├─ 可携带佣兵单位
├─ 佣兵有独立属性和数量
├─ 战斗中主将+佣兵视为一个单位
└─ 佣兵死亡可在营地补充

策略深度:
├─ 兵种搭配 (主将+佣兵组合)
├─ 数量分配 (质量vs数量)
├─ 经济投入 (培养vs更换)
└─ 战场策略 (佣兵肉盾/输出)

现代手游简化:
├─ 佣兵变为属性加成
├─ 或纯粹视觉表现
└─ 保留策略搭配概念
```

## 技能学习与升级

### 技能获取

```
获取方式:

等级解锁:
├─ 特定等级自动学会
├─ 职业进阶解锁
└─ 最传统的方式

技能树:
├─ 消耗技能点学习
├─ 前置技能要求
├─ 分支选择
└─ 无法学会所有技能

技能书/道具:
├─ 稀有技能需要特殊道具
├─ 可交易/掉落
└─ 经济系统一环

羁绊解锁:
├─ 特定角色组队解锁
├─ 剧情发展到某点
└─ 增加收集要素
```

### 技能升级

```
升级效果:

数值提升:
├─ 伤害增加
├─ 治疗量增加
├─ 持续时间延长
└─ 冷却时间减少

范围提升:
├─ 影响目标增加
├─ 作用范围扩大
└─ 射程增加

效果增强:
├─ 额外效果添加
├─ 负面效果减少
└─ 触发概率提升
```

## 羁绊/好感系统

```
《火焰纹章》支援系统:

好感度积累:
├─ 相邻作战
├─ 治疗/支援
├─ 共同出战
└─ 剧情互动

支援等级:
├─ C → B → A → S (部分作品)
├─ 每个等级有对话剧情
├─ 战斗加成递增
└─ 影响结局

战斗加成:
├─ 相邻时属性提升
├─ 概率协助攻击/防守
├─ 特定组合有额外效果
└─ 增加策略考量
```

## 长线养成规划

### 手游SRPG养成曲线

```
新手期 (Day 1-7):
├─ 快速升级体验
├─ 基础队伍成型
├─ 解锁核心系统
└─ 目标: 熟悉游戏

成长期 (Day 7-30):
├─ 角色培养深入
├─ 装备强化提升
├─ 挑战更高难度
└─ 目标: 建立主力队伍

成熟期 (Day 30+):
├─ 多队伍培养
├─ 追求极限数值
├─ 参与竞技/PVP
└─ 目标: 收集/竞技/社交
```

### 养成与关卡匹配

```
关键原则:

难度曲线:
├─ 关卡难度随玩家成长递增
├─ 养成必须能感知 (能打过之前打不过的)
├─ 避免突然难度跳变
└─ 提供不同难度选择

验证方式:
├─ 数值验证: 战力达标即可通过
├─ 策略验证: 需要正确打法
├─ 混合验证: 两者都需要
└─ 不同关卡侧重不同

内容消耗:
├─ 计算玩家成长速度
├─ 预留足够关卡内容
├─ 定期更新新内容
└─ 提供重复可玩内容
```

## 养成付费设计

```
手游常见模式:

角色获取:
├─ 抽卡获得新角色
├─ 碎片合成
└─ 活动/剧情赠送

加速养成:
├─ 体力购买 (更多关卡次数)
├─ 材料礼包
├─ 经验加成
└─ 立即完成

外观付费:
├─ 皮肤
├─ 特效
└─ 不影响数值

平衡要点:
- 付费有优势但非必须
- 策略技巧可以弥补数值差距
- 免费玩家有完整体验
- 付费玩家有性价比感
```

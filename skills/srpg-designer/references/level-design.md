# 关卡设计方法论 (Level Design)

## 关卡设计三维模型

### 基础框架

```
关卡由三个核心维度构成:

                    额外机制
                         │
           敌人强度 ─────┼───── 环境
                        / \
                       /   \
                      /     \

取点决定体验偏向:
- 偏向敌人 → 数值验证型
- 偏向机制 → 策略验证型  
- 偏向环境 → 空间利用型
```

#### 1. 敌人强度维度

```
敌人作为最直接的挑战:

数量:
├─ 少量精英 (高策略密度)
├─ 中等数量 (平衡型)
└─ 大量杂兵 (割草/持久战)

质量:
├─ 等级/属性数值
├─ 职业/兵种搭配
├─ AI行为模式
└─ 特殊技能组合

分布:
├─ 集中防守型
├─ 分散包围型
├─ 波次增援型
└─ 伏击埋伏型
```

#### 2. 环境维度

```
地形特征:

大小:
├─ 小型 (10x10) - 快速战斗
├─ 中型 (20x20) - 标准关卡
└─ 大型 (30x30+) - 史诗战斗

地形元素:
├─ 不可通行区域 (墙/悬崖)
├─ 障碍区域 (森林/水域)
├─ 高地/低地 (高低差)
└─ 特殊地形 (毒沼/火海)

路径设计:
├─ 单线推进
├─ 分路包围
├─ 中心交战区
└─ 环形迂回
```

#### 3. 额外机制维度

```
关卡特殊规则:

时间限制:
├─ 回合限制 (X回合内胜利)
├─ 波次计时 (定期增援)
└─ 环境变化 (特定回合触发)

机关陷阱:
├─ 可交互地形 (开关门/桥梁)
├─ 范围伤害 (落石/爆炸桶)
├─ 增益/减益区域
└─ NPC互动 (平民/友军)

特殊规则:
├─ 限制出战职业
├─ 禁止使用物品
├─ 强制携带特定角色
└─ 胜利条件变更
```

## 策略维度模型

### 深度设计

```
策略复杂度坐标系:

第三方机制 (道具/机关/环境互动)
              │
              │    ● 高复杂度
              │   / 
              │  /   (学习成本高，精通后成就感强)
              │ /
敌人机制 ─────┼───── 地形机制
(技能/职业)   │      (高低差/地形效果)
              │
              │
              ● 低复杂度
              (上手容易，但可能过于简单)

设计目标:
- 找到自己的定位
- 不同关卡可以调整位置
- 整体保持适度复杂度
```

### 敌人机制设计

```
敌人多样性:

职业多样性:
├─ 坦克型: 高防低移，吸引火力
├─ 输出型: 高攻低防，优先击杀
├─ 控制型: 限制移动/行动
├─ 辅助型: 治疗/增益敌人
└─ 特殊型: 独特机制 (飞行/隐形)

技能设计:
├─ 普攻: 标准攻击
├─ 强力技能: CD长/有代价
├─ 被动技能: 常驻效果
└─ 连携技能: 特定条件触发

AI行为:
├─ 进攻型: 主动接近玩家
├─ 防守型: 原地待命/守护区域
├─ 智能型: 根据局势变化
└─ 脚本型: 固定行为模式
```

### 地形机制设计

```
经典地形效果:

《火焰纹章》式:
├─ 森林: +20%闪避
├─ 山地: +10%防御
├─ 城堡: +2防御, +10%闪避
└─ 道路: 移动力+1

《皇家骑士团》式:
├─ 高低差: 影响射程和伤害
├─ 抛射线: 远程受高低差影响
├─ 悬崖: 跌落伤害/不可通过
└─ 水域: 特定职业可通行

《XCOM》式:
├─ 掩体: 提供防御加成
├─ 半掩/全掩: 不同防护等级
├─ 视线阻挡: 影响远程攻击
└─ 可破坏地形: 创造新路径
```

## 关卡类型设计

### 教学关卡

```
设计原则:
- 一次只教一个新概念
- 提供安全的试错环境
- 强制使用教学机制
- 及时正向反馈

示例流程:
关卡1-1: 基础移动和攻击
├─ 敌人: 1个弱鸡
├─ 地形: 平地
└─ 目标: 击败敌人

关卡1-2: 地形优势
├─ 敌人: 2个
├─ 地形: 森林 (闪避加成)
└─ 目标: 利用森林击败强敌

关卡1-3: 职业克制
├─ 敌人: 骑兵
├─ 玩家: 枪兵 (克制骑兵)
└─ 目标: 理解克制关系
```

### 标准战斗关卡

```
均衡设计:
- 中等数量敌人
- 多样地形利用
- 多种策略可行
- 适度挑战难度

设计检查清单:
□ 有明确的交战区域
□ 有侧翼/后路可走
□ 地形有价值但非必须
□ 不同职业都有发挥空间
```

### 特殊目标关卡

```
护送型:
设计要点:
├─ 被护送单位脆弱但有一定自保能力
├─ 路线选择 (安全但慢 vs 危险但快)
├─ 敌人会优先攻击护送目标
└─ 可以控制护送目标的行动

守点型:
设计要点:
├─ 多波次敌人进攻
├─ 防御设施可利用
├─ 资源/援军管理
└─ 崩溃点设计 (防线被突破)

斩首型:
设计要点:
├─ 强敌守护在深处
├─ 可以正面突破或绕后
├─ 击败首领即胜利
└─ 杂兵可以不理会
```

### 精英/Boss关卡

```
Boss战设计:

阶段设计:
阶段1 (100-70% HP):
├─ 熟悉基础攻击模式
├─ 召唤少量小怪
└─ 测试玩家输出能力

阶段2 (70-30% HP):
├─ 增加新技能
├─ 地形变化
└─ 策略要求提升

阶段3 (30%以下):
├─ 狂暴模式
├─ 高伤害高风险
└─ 决胜阶段

Boss机制示例:
- 范围大招需要分散站位
- 召唤物需要优先清理
- 弱点部位需要特定职业攻击
- 环境互动可以削弱Boss
```

## 情感曲线设计

### 节奏控制

```
单关卡情感曲线:

紧张度
 │    ╱╲      ╱高潮
 │   ╱  ╲    ╱
 │  ╱    ╲  ╱
 │ ╱      ╲╱
 │╱
 └──────────────────► 时间
   开局  发展  高潮  结局

设计技巧:
- 开局: 简单敌人建立信心
- 发展: 逐渐增加压力
- 高潮: 最强挑战/关键决策
- 结局: 收获/成就感
```

### 《三角战略》案例分析

```
经典关卡: 沃荷德城防御战

关卡背景:
- 大军压境，敌众我寡
- 有隐藏防御机关可用
- 使用机关需付出代价 (毁坏领地)
- 道德抉择影响剧情

地图设计:
- 22x22 日字型地图
- 阶梯状地形 (我方在上)
- 房屋和摊位提供策略空间
- 可冻结地板增加变数

敌人设计:
- Boss前4回合不行动 (制造紧张感)
- 波次增援 (第4、8回合)
- 后方刷新制造危机

第三方机制:
- 强力防御机关 (杀敌伤我)
- 情感抉择 (领地 vs 胜利)
- 多结局设计
```

## 手游SRPG关卡设计

### 碎片化适配

```
时长控制:
- 主线关卡: 10-15分钟
- 日常关卡: 3-5分钟
- 挑战关卡: 15-20分钟

简化设计:
- 地图不宜过大
- 敌人数量控制
- 一次性关卡 (无重复探索价值)
- 自动战斗友好
```

### 养成验证

```
关卡作为养成验证:

数值验证:
├─ 推荐战力系统
├─ 数值碾压 vs 技巧通关
└─ 不同付费层级体验

策略验证:
├─ 特定角色/职业优势
├─ 技能搭配验证
└─ 站位和走位技巧

每日关卡设计:
├─ 素材本: 快速获取资源
├─ 装备本: 特定装备掉落
├─ 经验本: 角色升级
└─ 特殊本: 限时挑战
```

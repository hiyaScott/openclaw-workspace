## 一、英雄技能设计模板

### 1.1 设计流程

```
Step 1: 确定定位
├── 角色类型: 输出/坦克/辅助/控制/全能
├── 职业: 战士/法师/弓手/刺客/治疗等
├── 稀有度: 普通/稀有/史诗/传说
└── 核心玩法: 什么情况下这个角色最强？

Step 2: 机制设计
├── 核心机制: 1-2个核心玩法机制
├── 被动技能: 天赋/特性
├── 主动技能: 2-4个主动技能
└── 限制条件: 冷却/消耗/触发条件

Step 3: 数值设计
├── 属性分配: 根据定位分配属性权重
├── 技能倍率: 参考速查表
├── 冷却/消耗: 与强度匹配
└── 成长曲线: 1级→满级的成长

Step 4: 平衡验证
├── 同类型对比: 与现有角色对比
├── 极端情况: 最佳/最差情况分析
├── 配合测试: 与其他角色的配合
└── 反制手段: 如何克制这个角色
```

### 1.2 技能设计检查清单

```
□ 定位清晰
   - 这个角色在队伍中扮演什么角色？
   - 与同定位角色相比有什么独特之处？

□ 机制独特
   - 核心机制是否新颖？
   - 是否与现有机制重复？

□ 数值合理
   - 技能倍率是否符合类型标准？
   - 冷却/消耗是否与强度匹配？

□ 成长感
   - 升级是否有明显成长？
   - 高等级是否有质变？

□ 操作空间
   - 高手与新手使用是否有差距？
   - 是否有进阶操作技巧？

□ 反制手段
   - 是否有有效的克制方式？
   - 是否过于强势/弱势？

□ 配合潜力
   - 能否与其他角色形成配合？
   - 是否有特定的阵容搭配？

□ 体验反馈
   - 使用时的爽快感如何？
   - 是否有清晰的反馈？
```

## 二、关卡设计检查清单

### 2.1 关卡设计三维检查

```
敌人维度:
□ 敌人数量是否合理？（5-15个）
□ 敌人类型是否多样？（近战/远程/精英/Boss）
□ 敌人配置是否有策略空间？（站位/兵种搭配）
□ 敌人强度是否匹配关卡进度？
□ 是否有特殊的敌人机制？

环境维度:
□ 地图大小是否合适？（15x15 - 25x25）
□ 地形是否多样？（平地/森林/山地/水域）
□ 高低差是否创造策略空间？
□ 是否有遮蔽物/掩体？
□ 出生点/目标点位置是否合理？

机制维度:
□ 是否有第三方机制？（油桶/机关/道具）
□ 机制是否与敌人/环境配合？
□ 机制是否易懂但深入？
□ 胜利条件是否清晰？
□ 失败条件是否合理？
```

### 2.2 关卡难度曲线检查

```
前1/3关卡:
□ 教学新机制
□ 敌人数量较少
□ 有明显的提示
□ 容错空间较大

中1/3关卡:
□ 机制组合
□ 敌人数量适中
□ 需要一定策略
□ 有多种解法

后1/3关卡:
□ 复杂机制
□ 敌人数量较多
□ 需要深度策略
□ 有挑战性但可完成
```

## 三、数值平衡验证流程

### 3.1 数值设计验证流程

```
阶段1: 理论验证
├── 参考标准值
├── 计算理论DPS
├── 对比同类角色
└── 调整基础数值

阶段2: 模拟测试
├── Excel模拟
├── 极端情况测试
├── 成长曲线验证
└── 调整成长数值

阶段3: 实际测试
├── 内部测试
├── 收集反馈
├── 数据分析
└── 最终调整
```

### 3.2 数值平衡检查清单

```
□ 伤害输出
   - 理论DPS是否与定位匹配？
   - 爆发伤害是否过高/过低？
   - 持续输出是否合理？

□ 生存能力
   - 生命值是否与职业匹配？
   - 防御力是否合理？
   - 是否有足够的生存手段？

□ 机动能力
   - 移动力是否符合职业？
   - 突进/位移技能是否合理？
   - 与其他角色的机动差距是否合适？

□ 控制能力
   - 控制时长是否合理？
   - 控制命中率是否平衡？
   - 是否有反制手段？

□ 辅助能力
   - 治疗量是否充足？
   - 增益效果是否合适？
   - 与其他辅助的差异是否明显？

□ 资源消耗
   - 技能消耗是否合理？
   - 冷却时间是否与强度匹配？
   - 资源恢复是否平衡？

□ 成长曲线
   - 前期体验是否流畅？
   - 中期是否有成长感？
   - 后期是否有过强/过弱？

□ 极端情况
   - 最佳情况是否过于强势？
   - 最差情况是否还能发挥作用？
   - 是否有明显的"垃圾时间"？
```

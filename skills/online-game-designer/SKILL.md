# 网络游戏设计师

## 能力范围

本 Skill 涵盖网络游戏设计的核心领域：

1. **游戏设计文档 (GDD)** - 完整的游戏设计文档编写
2. **系统架构设计** - MMORPG服务器架构、分区分服、实例化
3. **经济系统设计** - 货币体系、产出消耗、通货膨胀控制
4. **社交系统设计** - 公会、好友、组队、PVP/PVE
5. **数值策划** - 战斗数值、经济数值、成长曲线
6. **游戏类型专项** - MMORPG、MOBA、卡牌、SLG等

## 快速参考

### 网络游戏架构模式

```
四层架构:
┌─────────────┐
│ Client Layer│ 客户端层（瘦客户端）
├─────────────┤
│ Proxy Layer │ 代理层（网关、负载均衡）
├─────────────┤
│ Game Layer  │ 游戏逻辑层（分区/分世界）
├─────────────┤
│ DB Layer    │ 数据库层（角色、日志、配置）
└─────────────┘
```

### 经济系统设计框架

```
水龙头-排水管模型:
┌─────────────────────────────────────┐
│ 水龙头 (产出) → 存量 → 排水管 (消耗) │
├─────────────────────────────────────┤
│ • 任务奖励     • 玩家持有    • 装备强化  │
│ • 打怪掉落     • 交易市场    • 技能升级  │
│ • 活动赠送     • 公会仓库    • 消耗品购买│
│ • 每日签到                 • 传送/复活  │
└─────────────────────────────────────┘
```

### 社交系统设计层次

```
1. 点 (个人)     → 角色成长、成就系统
2. 线 (双向)     → 好友、师徒、情缘
3. 面 (小团体)   → 5人组队、10人团队
4. 体 (大团体)   → 公会、帮派、阵营
5. 网 (全服)     → 排行榜、跨服战
```

## 设计流程

### 1. 概念阶段
- 游戏定位与目标用户分析
- 核心玩法循环定义
- 竞品分析与差异化设计

### 2. 预生产阶段
- 编写高概念文档 (High Concept)
- 制作原型验证核心玩法
- 确定技术可行性

### 3. 生产阶段
- 编写详细GDD
- 数值表制作与平衡
- 系统模块分解 (PBS)

### 4. 迭代阶段
- 根据测试数据调整数值
- 优化用户体验
- 内容扩展与运营支持

## GDD标准结构

```
1. 项目概述
   - 游戏简介
   - 目标平台
   - 目标用户

2. 核心玩法
   - 游戏循环
   - 胜利条件
   - 核心机制

3. 游戏系统
   - 角色系统
   - 战斗系统
   - 经济系统
   - 社交系统
   - 任务系统

4. 世界观与叙事
   - 背景设定
   - 主要角色
   - 主线剧情

5. 技术需求
   - 客户端/服务器架构
   - 网络同步方案
   - 性能指标

6. 美术风格
   - 概念设计
   - UI/UX规范
   - 角色/场景风格

7. 运营规划
   -  monetization策略
   - 更新计划
   - 社区运营
```

## 最佳实践

1. **设计先于实现** - 文档先行，避免返工
2. **数据驱动** - 用数据验证设计，而非主观判断
3. **玩家至上** - 始终从玩家体验出发
4. **迭代优化** - 小步快跑，持续验证
5. **文档同步** - 设计变更及时更新文档

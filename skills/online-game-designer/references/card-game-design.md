# 卡牌游戏设计

## 概述

卡牌游戏（Card Game）包括TCG（集换式卡牌游戏）、CCG（收藏式卡牌游戏）和Deckbuilder（卡组构建游戏）。这类游戏的核心乐趣在于卡组构建、策略规划和卡牌之间的协同效应。

## 卡牌游戏类型

### 1. TCG/CCG (Trading/Collectible Card Game)

**代表游戏**: Magic: The Gathering、Hearthstone、Yu-Gi-Oh!

**核心特征**:
- 玩家预先构建卡组
- 卡牌收藏与交易
- 扩展包更新
- 竞技对战

**设计要点**:
```
卡组构建限制:
├─ 卡牌数量限制 (如30/40/60张)
├─ 单卡数量限制 (如每种最多2/3/4张)
├─ 稀有度限制 (如最多4张橙卡)
└─ 费用曲线平衡
```

### 2. Deckbuilder (卡组构建游戏)

**代表游戏**: Slay the Spire、Dominion、Monster Train

**核心特征**:
- 游戏过程中构建卡组
- Roguelike元素
- 单局循环
- PvE为主

**设计要点**:
```
构建流程:
├─ 起始卡组 (固定)
├─ 战斗/事件获得新卡
├─ 精简卡组 (移除弱卡)
├─ 遗物/道具协同
└─ 形成Combo
```

### 3. 混合类型

**代表游戏**: Gwent、Legends of Runeterra

**核心特征**:
- 简化资源系统
- 独特胜利条件
- 创新机制

## 核心机制设计

### 1. 资源系统

#### 费用系统 (Mana/Energy)
```
常见类型:

自动增长型 (炉石):
├─ 每回合自动+1水晶
├─ 上限10点
└─ 未用完的水晶不保留

资源积累型 (万智牌):
├─ 需要出牌到场上作为资源
├─ 不同颜色/类型
└─ 保留到下一回合

固定分配型 (游戏王):
├─ 每回合固定行动次数
├─ 无需费用
└─ 依赖手牌管理
```

#### 费用曲线设计
```
费用曲线 (Mana Curve):

理想分布:
┌─────────┬────────┬────────┐
│ 费用    │ 数量   │ 占比   │
├─────────┼────────┼────────┤
│ 0-1费   │ 6-8张  │ 20-25% │
│ 2费     │ 6-8张  │ 20-25% │
│ 3费     │ 5-6张  │ 15-20% │
│ 4费     │ 4-5张  │ 12-15% │
│ 5费+    │ 3-4张  │ 10-15% │
└─────────┴────────┴────────┘

设计原则:
- 前期牌多: 保证开局有动作
- 峰值在2-3费: 最活跃的阶段
- 后期牌少但强力: 终结手段
```

### 2. 卡牌结构

#### 卡牌要素
```
标准卡牌结构:

┌───────────────────┐
│ 费用 (左上角)      │
├───────────────────┤
│                   │
│     卡牌插画        │
│                   │
├───────────────────┤
│ 卡牌名称           │
├───────────────────┤
│ 类型/种族/职业     │
├───────────────────┤
│ 效果文本           │
│ (关键词说明)       │
├───────────────────┤
│ 攻击力 / 生命值    │
└───────────────────┘

数值位置规则:
- 费用: 左上角 (出牌条件)
- 攻击力: 左下角 (对怪伤害)
- 生命值: 右下角 (生存能力)
```

#### 卡牌类型
```
常见分类:

随从/生物卡:
├─ 拥有攻击力和生命值
├─ 留在场上持续作战
└─ 可被攻击/破坏

法术卡:
├─ 即时效果
├─ 不留在场上
└─ 一次性使用

装备/武器卡:
├─ 装备给角色/随从
├─ 提供加成效果
└─ 有耐久度限制

场地/环境卡:
├─ 影响全场
├─ 持续多回合
└─ 双方共享或独占
```

### 3. 战斗系统

#### 战斗模型
```
对怪伤害:
- 攻击力 vs 生命值
- 伤害结算后HP减少
- HP归零则破坏

对脸伤害 (直接伤害):
- 直接减少玩家生命
- 通常攻击力 = 伤害值
- 部分游戏有护甲/护盾机制

交换计算:
┌─────────────────────────────────────┐
│ 攻击方: 4/4 (攻击力/生命值)          │
│ 防守方: 3/5                         │
│                                     │
│ 结果:                               │
│ - 防守方受到4点伤害 → 3/1           │
│ - 攻击方受到3点伤害 → 4/1           │
│ (双方同时攻击)                      │
└─────────────────────────────────────┘
```

#### 战斗关键词
```
常见关键词:

冲锋/疾走: 上场即可攻击
嘲讽/守护: 必须优先攻击
潜行/隐匿: 无法被指定攻击
吸血/吸血: 造成伤害时恢复生命
圣盾/护盾: 免疫一次伤害
亡语/遗言: 死亡时触发效果
战吼/入场: 上场时触发效果
连击/连击: 本回合出过牌后获得加成
```

## 卡牌设计原则

### 1. 卡牌平衡

#### 费用-效果比
```
白模 (Vanilla) 设计:

1费: 1/1 或 2/1
2费: 2/3 或 3/2
3费: 3/4 或 4/3
4费: 4/5 或 5/4
5费: 5/6 或 6/5

带效果的卡牌:
- 效果价值 = 费用的一部分
- 身材相应减小
- 例: 2费 2/2 + 抽一张牌
```

#### 权重评分法
```
《游戏王》GENESYS赛制示例:

每张卡牌赋予积分值:
├─ 灰流丽: 15分
├─ 深渊潜伏者: 100分
└─ 卡组构建预算: 100分

设计优势:
- 强制卡组强度收敛
- 保留策略多样性
- 数学化平衡手段
```

### 2. 卡牌稀有度设计

```
稀有度与强度关系:

普通 (Common):
├─ 基础效果
├─ 卡组填充
└─ 易获得

稀有 (Rare):
├─ 强力效果
├─ 构筑核心
└─ 适度稀缺

史诗 (Epic):
├─ 独特机制
├─ 卡组灵魂
└─ 较难获得

传说 (Legendary):
├─ 独一无二的机制
├─ 限制数量 (通常1张)
└─ 极度稀缺

设计原则:
- 稀有度 ≠ 绝对强度
- 高稀有度 = 独特/有趣
- 低稀有度卡也有存在价值
```

### 3. 构筑模型

#### 费用曲线模型
```
卡组构筑检查清单:

□ 前期 (1-3费): 足够的出牌
□ 中期 (4-6费): 节奏控制点
□ 后期 (7费+): 终结手段
□ 费用平滑: 每回合有事可做
□ 功能齐全: 解场/站场/过牌
```

#### 卡组类型
```
快攻 (Aggro):
├─ 低费牌多 (60%+是1-3费)
├─ 曲线偏低
├─ 强调前期压力
└─ 快速结束战斗

中速 (Midrange):
├─ 均衡的费用分布
├─ 质量生物为主
├─ 按费拍怪
└─ 稳定输出

控制 (Control):
├─ 高费牌多
├─ 大量解牌
├─ 苟活到后期
└─ 大后期终结

组合技 (Combo):
├─ 特定卡牌配合
├─ 过牌/检索多
├─ 前期生存
└─ Combo完成即胜利
```

## Deckbuilder 特殊设计

### 1. 循环机制
```
Slay the Spire 设计:

牌库循环:
├─ 抽完牌库后弃牌堆洗牌
├─ 循环使用卡牌
├─ 鼓励精简卡组
└─ 关键卡反复使用

策略深度:
- 卡组越精简, 关键牌出现频率越高
- 需要平衡"加卡"与"删卡"
- 遗物与卡牌的协同
```

### 2. 卡牌协同
```
协同类型:

直接协同:
├─ A卡增强B卡效果
├─ 例: "每当打出攻击牌, 获得1点护甲"
└─ 明确的配合关系

间接协同:
├─ 共享资源/机制
├─ 例: 大量弃牌牌 + 弃牌触发效果
└─ 系统性的配合

条件协同:
├─ 满足特定条件触发
├─ 例: 手牌为空时获得加成
└─ 需要刻意营造条件
```

### 3. 风险与收益
```
卡牌设计平衡:

高风险高回报:
├─ 强力效果但有限制
├─ 例: 造成10点伤害, 但消耗所有能量
└─ 需要时机把握

稳定收益:
├─ 可靠但上限较低
├─ 例: 造成3-5点伤害
└─ 适合日常过渡

资源转换:
├─ 资源A → 资源B
├─ 例: 失去5生命, 抽3张牌
└─ 策略性选择
```

## 数值设计

### 1. 伤害数值
```
有效数位设计:

日系TCG (千位制):
├─ 青眼白龙: 3000攻击力
├─ 黑魔导: 2500攻击力
└─ 原因: 数值大显强, 切分粒度足够

西式TCG (小数值):
├─ 炉石: 1-12攻击力为主
├─ 万智牌: 1-10攻击力为主
└─ 原因: 计算简便, 数字有意义

对脸伤害:
├─ 生命护甲制: 1-3点伤害
├─ 血量制: 根据总血量调整
└─ 需要可预期的击杀回合数
```

### 2. 抽牌与资源
```
手牌管理:

起始手牌: 3-4张
每回合抽牌: 1-2张
手牌上限: 8-10张

过牌价值:
- 抽1张牌 ≈ 0.5-1费的价值
- 大量过牌需要代价
- 滤牌(抽牌+弃牌)价值略低
```

## 游戏案例分析

### 案例1: Hearthstone (炉石传说)
```
成功要素:
├─ 简化万智牌规则
├─ 每回合自动增长水晶
├─ 明确的职业特色
├─ 丰富的随机性
└─ 易上手难精通

设计特点:
- 法术力系统简化
- 随从交换为核心
- 英雄技能提供保底
- 扩展包持续更新
```

### 案例2: Slay the Spire (杀戮尖塔)
```
成功要素:
├─ Roguelike + Deckbuilder
├─ 战斗中无法重组卡组
├─ 遗物系统增加变数
├─ 三个职业不同机制
└─ 极高的重玩价值

设计特点:
- 卡组构建在游戏过程中
- 卡牌+遗物协同
- 意图系统降低不确定性
- 策略重于操作
```

### 案例3: Magic: The Gathering (万智牌)
```
成功要素:
├─ 五色系统深度
├─ 堆叠机制复杂度
├─ 20+年持续更新
├─ 竞技与休闲并存
└─ 收藏属性

设计特点:
- 资源系统深度最高
- 互动性极强
- 时机选择重要
- 颜色轮哲学
```

## UI/UX设计

### 1. 卡牌可读性
```
设计原则:

清晰优先:
├─ 大字体费用数字
├─ 明确的攻防标识
├─ 关键词突出显示
└─ 插画不遮挡重要信息

一致性:
├─ 同类卡牌布局统一
├─ 颜色编码 (费用/类型)
├─ 图标标准化
└─ 提示信息规范
```

### 2. 交互设计
```
操作反馈:
├─ 拖拽卡牌的手感
├─ 目标选择的明确性
├─ 效果触发的视觉反馈
└─ 撤销/回退机制

信息展示:
├─ 卡牌详情查看
├─ 历史记录回溯
├─ 对手信息展示
└─ 计时器/回合提示
```

# 数值策划方法

## 概述

数值策划是游戏设计中连接设计与实现的桥梁。优秀的数值系统让玩家感受到公平、有成就感和持续挑战。

## 数值设计原则

### 1. 可预期性
- 玩家能理解数值变化
- 因果关系清晰
- 避免黑箱随机

### 2. 可比较性
- 不同选择间可以对比
- 决策有意义
- 避免"假选择"

### 3. 可扩展性
- 数值系统能支撑长期更新
- 预留增长空间
- 避免数值爆炸

## 战斗数值

### 基础属性设计

#### 属性类型
```
一级属性 (基础):
├─ 力量 (影响物理攻击)
├─ 敏捷 (影响攻速/闪避)
├─ 智力 (影响魔法攻击)
└─ 体质 (影响生命值)

二级属性 (战斗):
├─ 攻击力
├─ 防御力
├─ 生命值
├─ 攻击速度
└─ 暴击率/暴击伤害
```

#### 属性权重
```
权重计算:

1点力量 = 2点物理攻击
1点敏捷 = 1点攻速 + 0.5%闪避
1点智力 = 2.5点魔法攻击
1点体质 = 10点生命值

权重决定属性价值，影响装备和技能设计
```

### 伤害公式

#### 基础公式
```
伤害 = 攻击力 × 技能倍率 × (1 - 防御减免)

防御减免 = 防御力 / (防御力 + 常数K)

K值决定防御收益曲线:
- K值大 → 防御收益递减慢
- K值小 → 防御收益递减快
```

#### 进阶公式
```
最终伤害 = (基础攻击 + 额外攻击) 
         × 技能倍率 
         × 暴击倍率(如果暴击)
         × 元素倍率
         × (1 - 目标防御减免)
         × (1 + 伤害加成)
         × 随机浮动(0.95-1.05)
```

### 战斗平衡

#### TTK (Time To Kill) 控制
```
TTK设计:

PvE:
├─ 普通怪: 3-5秒
├─ 精英怪: 10-20秒
└─ BOSS: 2-5分钟

PvP:
├─ 快速击杀: 5-10秒 (高爆发职业)
├─ 标准战斗: 15-30秒
└─ 持久战: 1-2分钟 (坦克vs坦克)
```

#### 职业平衡
```
三角平衡模型:

坦克:
├─ 高防御/生命
├─ 低输出
└─ 控制技能

输出:
├─ 高攻击
├─ 低防御
└─ 爆发/持续

治疗:
├─ 恢复能力
├─ 辅助技能
└─ 生存能力中

克制关系:
输出 > 治疗 > 坦克 > 输出
(理想情况)
```

## 经济数值

### 价值计算

#### 基准价值
```
价值锚定:

1. 确定时间基准
   玩家1小时游戏 = X 钻石 = Y RMB

2. 计算单位价值
   1分钟 = X/60 钻石

3. 资源价值计算
   资源价值 = 获取时间 × 单位时间价值 × 难度系数
```

#### 产出价值表
```
产出设计:

活动        时间    产出价值    难度系数
─────────────────────────────────────
日常任务    30min   50钻        1.0
副本挑战    15min   40钻        1.2
世界BOSS    10min   100钻       2.0
PVP竞技     20min   60钻        1.5

难度系数考虑:
├─ 战斗难度
├─ 组队要求
├─ 时间窗口
└─ 竞争程度
```

### 消耗设计

#### 消耗曲线
```
成长消耗曲线:

等级    升级消耗    累计消耗
─────────────────────────────
1-10    100/级      1,000
11-20   500/级      6,000
21-30   2000/级     26,000
31-40   5000/级     76,000

曲线公式:
消耗 = 基础值 × (等级^指数)
指数通常: 1.5-2.5
```

#### 消耗类型比例
```
消耗分配:

假设每日产出: 1000金币

装备强化:     40% (400金币)
技能升级:     25% (250金币)
药水消耗:     20% (200金币)
其他:         15% (150金币)

比例根据游戏类型调整
```

## 成长数值

### 经验曲线

#### 曲线类型
```
经验曲线设计:

线性增长:
├─ 每级经验固定增加
├─ 简单直观
└─ 后期升级过快

指数增长:
├─ 每级经验指数增加
├─ 控制成长节奏
└─ 标准MMO设计

自定义曲线:
├─ 前期快后期慢
├─ 特定等级调整
└─ 配合内容投放
```

#### 经验公式
```
指数曲线公式:

所需经验 = 基础经验 × (等级 ^ 指数)

示例:
基础经验 = 100
指数 = 2.5

1级:  100 × 1^2.5   = 100
10级: 100 × 10^2.5  = 31,623
50级: 100 × 50^2.5  = 1,767,767

总经验需求从1到N级:
总经验 = Σ(基础经验 × i^指数), i=1 to N
```

### 升级时间规划

#### 时间曲线
```
升级时间设计:

等级段      每级时间      设计目的
───────────────────────────────────
1-10级      5-10分钟     快速入门
11-20级     15-30分钟    熟悉系统
21-30级     1-2小时      核心体验
31-40级     3-5小时      深度内容
41-50级     1天          长期目标
```

### 战力成长

#### 战力公式
```
战力计算:

基础战力 = 攻击 × 系数1 + 防御 × 系数2 + 生命 × 系数3

实际战力 = 基础战力 × (1 + 技能加成 + 套装加成 + 其他加成)

系数确定:
根据DPS和生存能力平衡确定
```

#### 战力曲线
```
战力成长曲线:

战力
 │
 │    ╭────────── 付费玩家
 │   ╱
 │  ╱    ╭────── 活跃玩家
 │ ╱    ╱
 │╱    ╱   ╭──── 休闲玩家
 └───────────────────────► 时间/投入

差距控制:
付费vs免费战力比控制在2-3倍内
```

## 概率设计

### 随机分布

#### 分布类型
```
常用分布:

均匀分布:
├─ 所有结果概率相等
├─ 示例: 骰子
└─ 适用: 基础随机

正态分布:
├─ 中间值概率高
├─ 极端值概率低
└─ 适用: 属性浮动

泊松分布:
├─ 稀有事件发生
├─ 适合掉落率
└─ 适用: 稀有物品掉落
```

#### 伪随机分布
```
PRD (Pseudo Random Distribution):

机制:
每次不成功增加概率，直到成功

示例 (20%暴击率):
├─ 第1次: 5.6%
├─ 第2次: 11.2%
├─ 第3次: 16.8%
├─ 第4次: 22.4%
└─ ...直到暴击

优点: 避免长期不暴击的负面体验
```

### 抽卡设计

#### 概率模型
```
抽卡概率设计:

基础概率:
├─ SSR: 1%
├─ SR:  10%
├─ R:   89%

保底机制:
├─ 软保底: 50抽后概率递增
├─ 硬保底: 90抽必出SSR
└─ 大保底: 180抽必出UP角色

期望值计算:
E = Σ(概率i × 价值i)
```

## 数值验证

### 验证清单

#### 战斗验证
```
□ TTK是否符合设计
□ 各职业DPS是否平衡
□ 坦克生存时间是否合理
□ PvP战斗时长是否合适
□ 不同等级差距是否合理
```

#### 经济验证
```
□ 每日产出是否满足消耗
□ 通货膨胀率是否可控
□ 付费转化率是否合理
□ 玩家生命周期价值
□ 不同付费层级差距
```

#### 成长验证
```
□ 升级时间曲线是否合理
□ 战力成长是否平滑
□ 内容消耗速度是否匹配
□ 留存率是否符合预期
```

# QA 测试工程

## 概述

QA（Quality Assurance）测试工程是软件开发生命周期中的关键环节，涵盖从需求分析到发布验证的全流程质量保障。本技能整合自动化代码检查与手动测试执行能力，提供完整的测试解决方案。

## 核心能力

### 1. 测试规划
- **测试计划制定**：根据项目需求制定测试策略和计划
- **测试用例设计**：功能测试、边界测试、异常测试用例编写
- **测试范围界定**：确定测试优先级和覆盖范围

### 2. 自动化代码检查

静态分析游戏代码，检测常见问题：

| 检查项 | 说明 |
|-------|------|
| **HTML结构** | 检查DOCTYPE声明、基本HTML结构 |
| **Canvas初始化** | 检测2D上下文是否正确获取 |
| **游戏循环** | 检查requestAnimationFrame是否存在 |
| **事件监听** | 检测键盘/鼠标事件监听 |
| **音频系统** | 检查AudioContext配置 |
| **代码安全** | 检测未定义变量风险、无限循环风险 |

**检查结果级别**：
- ✅ **通过** - 无明显代码问题
- ⚠️ **警告** - 潜在问题需人工判断
- ❌ **失败** - 严重问题需立即修复

### 3. 手动测试执行

#### 核心测试项

| 测试项 | 说明 |
|-------|------|
| **加载测试** | 游戏能否正常加载，无报错 |
| **第一关验证** | 第一关能否正常开始和完成 |
| **控制响应** | 键盘/鼠标/触摸是否响应 |
| **游戏逻辑** | 核心玩法逻辑是否正常 |
| **音频测试** | 音效和背景音乐是否正常播放 |
| **视觉检查** | UI显示、动画、特效是否正常 |

#### 测试清单

- [ ] 游戏能正常加载
- [ ] 第一关正常开始
- [ ] 控制响应及时
- [ ] 游戏逻辑正常
- [ ] 能完成第一关
- [ ] 音频正常播放
- [ ] 无明显视觉问题
- [ ] 跨浏览器兼容

### 4. Bug跟踪与管理
- **Bug报告编写**：复现步骤、预期结果、实际结果、环境信息
- **严重程度分级**：致命/严重/一般/轻微
- **回归测试**：修复验证和回归测试执行

## 测试类型

### 功能测试
验证软件功能是否符合需求规格说明。

### 兼容性测试
- 跨浏览器测试（Chrome、Firefox、Safari、Edge）
- 跨设备测试（桌面、平板、手机）
- 跨操作系统测试

### 性能测试
- 加载时间测试
- 帧率稳定性测试
- 内存占用监控

### 用户体验测试
- UI/UX一致性检查
- 交互流畅度评估
- 可访问性测试

## 工具与方法

### 常用工具
- **Bug管理**：Jira、Trello、GitHub Issues
- **测试用例**：TestRail、Excel、Markdown
- **自动化测试**：Selenium、Playwright、Jest
- **性能分析**：Chrome DevTools、Lighthouse

### 最佳实践
1. **尽早测试** - 在开发早期介入测试
2. **持续集成** - 将测试集成到CI/CD流程
3. **文档化** - 详细记录测试用例和结果
4. **可追溯性** - 确保需求-用例-Bug的关联

## 注意事项

- 部分游戏需要用户交互后才能播放音频（浏览器策略限制）
- 回合制游戏与实时游戏测试重点不同
- Canvas游戏与DOM游戏检查方式不同
- 自动检查不能替代人工测试，仅作为初步筛查

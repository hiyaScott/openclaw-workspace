# GRID.DOMINION - 区域控制Roguelike游戏设计文档

> **版本**: 1.0  
> **类型**: 快节奏Roguelike动作游戏  
> **视觉风格**: 代码图形/极简几何

---

## 1. 游戏名称和世界观

### 1.1 游戏名称
**GRID.DOMINION**（网格统治）

### 1.2 世界观设定

**背景故事**  
在数字世界的边缘，存在一个由纯代码构成的维度——**The Grid**。这里的一切都由0和1组成，空间被划分为规则的网格单元。玩家扮演一个**觉醒的程序片段（Rogue Process）**，获得了在网格中"染色"的能力。

The Grid被**系统守护进程（System Daemons）**控制，它们不断清除异常代码。玩家必须通过占领网格区域来扩展自己的存在空间，同时抵御守护进程的清除行动。

**视觉风格：代码图形（Code-Graphic）**
- 网格地板：深色背景上的发光线条
- 玩家：简洁的几何形状（三角形/菱形），带有脉动光效
- 敌人：几何图形组合，带有警告色（红/橙）
- 已占领区域：玩家主题色填充，带有微妙的代码雨效果
- UI：终端风格，等宽字体，简洁的进度条

**核心隐喻**
- 占领 = 代码执行/感染
- 敌人 = 杀毒程序/系统进程
- 死亡 = 进程被终止
- 升级 = 代码优化/功能解锁

## 2. 核心玩法循环详细规则

### 2.1 基础操作

| 操作 | 输入 | 效果 |
|------|------|------|
| 移动 | WASD / 方向键 / 摇杆 | 向相邻格子移动 |
| 冲刺 | Shift / 空格 | 快速移动3格，冷却3秒 |
| 技能 | Q/E/R | 使用装备的技能 |

### 2.2 占领机制

**基础占领**
- 玩家移动到未占领格子时，自动占领该格
- 占领的格子变为玩家主题色，产生微弱光效
- 已占领格子为玩家的"安全区域"

**领地伤害（核心机制）**
- 敌人进入玩家领地时，每秒受到 **领地等级 × 5** 点伤害
- 伤害数字在敌人头顶显示
- 敌人在领地内移动速度降低 **20%**

**领地等级系统**
| 等级 | 条件 | 伤害加成 | 效果 |
|------|------|----------|------|
| Lv.1 | 基础 | 5/秒 | 无 |
| Lv.2 | 占领>30% | 8/秒 | 敌人减速30% |
| Lv.3 | 占领>50% | 12/秒 | 玩家移速+10% |
| Lv.4 | 占领>70% | 15/秒 | 敌人受到眩晕效果 |

### 2.3 通关条件

**房间通关**
- 占领率达到 **80%** 时，房间通关
- 显示"DOMINION ACHIEVED"，开启出口
- 进入出口进入下一层/房间

**失败条件**
- 玩家生命值归零
- 被敌人逼入死角无法移动（5秒内无有效移动 → 死亡）

### 2.4 节奏控制

**时间压力系统**
- 每过 **30秒**，生成一波新的敌人增援
- 敌人数量随时间递增
- 未占领区域会随时间缓慢"腐蚀"（变为危险地形）

**动态难度**
- 根据玩家当前占领率调整敌人生成速度
- 占领率<30%：敌人生成速度+50%
- 占领率30-60%：正常生成
- 占领率>60%：敌人生成速度-30%

## 3. 房间类型和地图生成算法

### 3.1 房间类型

| 类型 | 尺寸 | 特点 | 敌人密度 |
|------|------|------|----------|
| **标准房间** | 12×12 | 无障碍，适合练习 | 中 |
| **迷宫房间** | 15×15 | 墙壁分隔，需要绕路 | 高 |
| **开放大厅** | 20×20 | 空间大，敌人多 | 极高 |
| **狭窄走廊** | 8×20 | 长条形，战术要求高 | 中 |
| **中心广场** | 16×16 | 中心有障碍物 | 高 |
| **Boss房间** | 18×18 | 特殊地形，精英敌人 | Boss+护卫 |

### 3.2 地图生成算法

**算法：分块随机生成 + 连通性保证**

```
生成流程：
1. 确定房间类型和基础尺寸
2. 生成基础网格（全部未占领）
3. 根据类型放置障碍物
   - 迷宫：随机DFS生成墙壁
   - 大厅：中心放置大型障碍物
   - 走廊：两侧生成墙壁
4. 确保玩家出生点周围3×3区域无障碍
5. 确保至少存在一条从起点到任意位置的路径
6. 随机放置初始敌人（远离玩家）
7. 生成出口位置（距离起点最远的可达位置）
```

**障碍物类型**
| 类型 | 效果 |
|------|------|
| 墙壁 | 不可通过，阻挡视线 |
| 陷阱格 | 玩家经过时受到伤害 |
| 减速区 | 移动速度-50% |
| 传送点 | 接触后随机传送到另一个传送点 |

### 3.3 关卡结构

**Run结构**
- 每个Run包含 **5个普通房间 + 1个Boss房间**
- 房间之间通过短走廊连接
- 每2个房间后提供一次商店/休息机会

**房间奖励**
- 通关时根据剩余时间和占领速度给予评分
- 评分等级：S/A/B/C/D
- S级：额外奖励选择 + 稀有道具
- A级：标准奖励选择
- B级及以下：基础奖励

## 4. 敌人种类和行为AI

### 4.1 基础敌人

#### **SCANNER（扫描器）**
```
外观：圆形，带有扫描线效果
生命值：30
速度：中等
行为：
  - 直线朝玩家移动
  - 进入领地后犹豫0.5秒再进入
  - 简单寻路，遇到障碍会绕路
AI状态机：
  IDLE → CHASE(发现玩家) → ATTACK(接触玩家) → RETREAT(血量<20%)
```

#### **PURSUER（追击者）**
```
外观：三角形，指向玩家方向
生命值：50
速度：快
行为：
  - 始终追踪玩家位置
  - 领地内受到伤害后加速逃离
  - 会尝试包抄玩家
特殊：在领地外时速度+30%
```

#### **BLOCKER（阻挡者）**
```
外观：方形，厚重感
生命值：100
速度：慢
行为：
  - 移动到玩家前方阻挡路径
  - 会占据关键位置（狭窄通道）
  - 领地内几乎不动，成为"路障"
特殊：领地内伤害减免50%
```

### 4.2 进阶敌人

#### **SWARM（集群）**
```
外观：小型六边形，成群出现
生命值：10（每个）
速度：快
行为：
  - 3-5个一组生成
  - 分散包围玩家
  - 领地内快速死亡，但数量多
特殊：死亡时短暂提升附近同伴速度
```

#### **SNIPER（狙击手）**
```
外观：菱形，带有瞄准线
生命值：40
速度：慢
行为：
  - 保持与玩家距离
  - 定期发射投射物
  - 优先站在未占领区域
特殊：投射物可以"净化"已占领格子（变回未占领）
```

#### **DASHER（冲刺者）**
```
外观：箭头形状，拖尾效果
生命值：35
速度：极快（冲刺时）
行为：
  - 定期向玩家方向冲刺
  - 冲刺路径上的格子被"污染"（暂时无法占领）
  - 冲刺后需要2秒恢复
```

### 4.3 Boss敌人

#### **OVERSEER（监视者）**
```
外观：大型几何体组合，多阶段变形
生命值：500
阶段1（100-70%血量）：
  - 召唤SCANNER和PURSUER
  - 定期释放全屏冲击波
阶段2（70-30%血量）：
  - 新增召唤BLOCKER
  - 冲击波频率增加
  - 随机"净化"一片已占领区域
阶段3（30-0%血量）：
  - 召唤所有类型敌人
  - 移动速度大幅提升
  - 接触玩家造成巨额伤害
特殊机制：
  - 必须占领80%区域才能对其造成有效伤害
  - 否则伤害减免90%
```

### 4.4 AI行为系统

**通用AI架构**
```
EnemyAI:
  - Perception: 检测玩家位置、领地范围
  - Decision: 基于当前状态选择行为
  - Action: 执行移动/攻击/技能
  
状态机：
  IDLE: 巡逻/待机
  ALERT: 发现玩家，准备行动
  CHASE: 追踪玩家
  ATTACK: 执行攻击
  RETREAT: 逃离领地/恢复
  DEAD: 死亡处理
```

**寻路算法**
- A*寻路，考虑障碍物
- 领地权重：敌人在路径规划时会"避开"高等级领地
- 动态更新：每0.5秒重新评估路径

## 7. 技术实现建议（AI编程友好）

### 7.1 技术栈推荐

**推荐方案：Godot 4.x + GDScript**
- 轻量级，适合2D网格游戏
- 内置A*寻路
- 节点系统适合实体管理
- 导出Web/PC/移动端

**备选方案**
- Unity + C#（功能更丰富，但较重）
- Pygame（快速原型，性能一般）
- JavaScript + Canvas（Web优先）

### 7.2 代码架构

**核心类设计**
```gdscript
# 游戏主控制器
class GameManager:
    - grid: GridSystem
    - player: Player
    - enemies: Array[Enemy]
    - current_room: Room
    - run_stats: RunStats

# 网格系统
class GridSystem:
    - width, height: int
    - cells: Array[CellState]
    - get_cell(pos): CellState
    - occupy(pos, owner): void
    - get_occupancy_rate(): float

# 格子状态
enum CellState:
    EMPTY      # 未占领
    OWNED      # 玩家占领
    ENEMY      # 敌人占领（Boss战）
    BLOCKED    # 障碍物
    CORRUPTED  # 腐蚀/危险

# 玩家
class Player:
    - position: Vector2i
    - health: int
    - skills: Array[Skill]
    - move(direction): bool
    - use_skill(index): void

# 敌人基类
class Enemy:
    - position: Vector2i
    - health: int
    - state: AIState
    - update(delta): void
    - take_damage(amount): void
    - on_enter_owned_cell(): void  # 进入玩家领地
```

### 7.3 关键算法实现

**领地伤害检测**
```gdscript
func _process(delta):
    for enemy in enemies:
        if grid.get_cell(enemy.position) == CellState.OWNED:
            var damage = base_territory_damage * territory_level * delta
            enemy.take_damage(damage)
```

**A*寻路简化版**
```gdscript
func find_path(start, end) -> Array[Vector2i]:
    # 使用Godot内置A*或自定义实现
    # 考虑领地权重：敌人倾向于避开玩家领地
    var astar = AStarGrid2D.new()
    astar.region = Rect2i(0, 0, width, height)
    astar.update()
    
    # 设置领地格子的权重
    for pos in owned_cells:
        astar.set_point_weight_scale(pos, 10.0)  # 高权重，敌人不喜欢走
    
    return astar.get_point_path(start, end)
```

**房间生成**
```gdscript
func generate_room(type: RoomType) -> Room:
    var room = Room.new()
    room.setup_dimensions(type)
    
    match type:
        RoomType.STANDARD:
            # 简单空地
            pass
        RoomType.MAZE:
            generate_maze_walls(room)
        RoomType.OPEN_HALL:
            place_center_obstacle(room)
    
    ensure_connectivity(room)
    place_initial_enemies(room)
    place_exit(room)
    
    return room
```

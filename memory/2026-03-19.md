## 认知负载监控 v6.0 架构简化 (Session 3)

**时间**: 19:00-19:25  
**目标**: 基于"30-60秒延迟可接受"简化整个监控系统  
**状态**: ✅ 已完成

### 核心决策

用户明确表示："30秒、60秒的延迟，完全没有问题"  
这允许我们从复杂的实时架构（Redis + 三级回退）简化为纯 GitHub Pages 方案。

### 架构对比

| 维度 | v5.41 | v6.0 |
|------|-------|------|
| **数据源** | Redis → CDN → GitHub Raw | 只用 GitHub Raw |
| **轮询间隔** | 10秒 | 30秒 |
| **评分算法** | 120行复杂计算 (8因素) | 30行简单映射 |
| **预估时间** | ❌ | ✅ |
| **代码行数** | ~370行 | ~120行 |
| **依赖** | Redis API, CDN | 无 (纯 GitHub) |

### 简化后的评分逻辑

```python
队列长度 = 处理中 + 等待中

if 队列长度 == 0: 评分 = 0%   # 空闲
if 队列长度 == 1: 评分 = 25%  # 轻载
if 队列长度 == 2: 评分 = 50%  # 中等
if 队列长度 >= 3: 评分 = 75%+ # 高载
```

### 预估回复时间

```
队列长度 0 → "立即"
队列长度 1 → "~30秒"
队列长度 2 → "~1分钟"
队列长度 3+ → "~N分钟"
```

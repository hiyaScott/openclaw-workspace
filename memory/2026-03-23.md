### 为什么没有清理定时任务

保留了两个 crontab 任务:
1. `cognitive_monitor.py` (每分钟) - 数据收集，写入本地文件
2. `cognitive_push_v6.sh` (每5分钟) - 备份到 GitHub

它们只是**备份机制**，API 服务器直接从本地文件读取，不依赖 GitHub。

### 为什么之前弃用了 API 方案？

**历史回顾** (v5.38/v5.39 时期):

我们确实实现过 API 服务器，但后来好像没有持续使用。可能的原因:

1. **服务器稳定性** - API 服务器需要保持运行，如果崩溃需要自动重启
2. **网络暴露** - 需要开放端口，可能有安全考虑
3. **复杂性** - 当时觉得 CDN 方案"够用"，增加了维护负担
4. **CDN 缓存问题** - 当时没有意识到 CDN 延迟有多严重

**现在的不同**:
- 使用 `nohup` 保持 API 服务器后台运行
- 使用非特权端口 8080
- 用户明确表达了实时性的需求
- 体验过 CDN 的 15+ 分钟延迟后，API 的优势很明显

### 架构总结

```
用户 ←→ API (实时, 5秒)
       ↓
   本地文件 ←← 监控脚本 (每分钟)
       ↓
   GitHub (备份, 每5分钟)
       ↓
   CDN (回退)
```

**实时路径**: API → 本地文件 (毫秒级)
**备份路径**: 本地文件 → GitHub → CDN (5-15分钟)

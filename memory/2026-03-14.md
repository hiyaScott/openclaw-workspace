## 编钟模拟器 MIDI 导入逻辑修复

**时间**: 2026-03-14 (预压缩内存刷新)
**版本**: v1.8.9
**提交**: 71f1e8f

### 问题
用户导入的 MIDI 文件保留了原始音高（如 C#、F# 等半音），但编钟模拟器内置的 `convertMidiToBells()` 函数会强制做以下转换：
1. C1区域音符 → 琶音序列（非钢琴音域）
2. 半音 → 映射到最近五声音阶音
3. 音域限制 C3-C6 → 范围外的音符被丢弃
4. 时间戳重对齐 → 原始时间信息丢失

### 解决方案
创建新的 `convertMidiToBellsStrict()` 严格模式函数：
- 保留所有音符，不做琶音转换
- 扩展音域到 C2-C7 (MIDI 36-96)
- 保留原始音高，不做五声音阶映射
- 保留原始时间戳

### 删除的内置曲目
由于原内置曲目（难念的经、倩女幽魂、青花瓷）包含大量琶音转换标记 `isArpeggio: true`，数据不准确，已清空内置曲目列表。

### 用户操作建议
用户现在可以通过 MIDI 导入功能添加准确的曲目，导入后的数据将：
1. 保留所有原始音符
2. 保留原始音高
3. 保留原始时间戳

## 3D打印技能页面修复 (2026-03-22 14:30)

### 问题报告
用户反馈 `kimi-claw/bambu-3dprint/` 页面的聊天界面无法使用：
1. 输入文字无法发送
2. 点击场景卡片无响应
3. 快速问题按钮无效

### 根本原因
JavaScript 使用了 ES6 语法，在某些浏览器/设备上不兼容：
- 箭头函数 `() => {}`
- 模板字符串 `` ` ``
- `const/let` 声明
- `for...of` 循环
- `includes()` 方法

### 修复方案
将代码改写为 ES5 兼容语法：
- `const` → `var`
- 箭头函数 → `function() {}`
- 模板字符串 → 普通字符串 + `<br>`
- `includes()` → `indexOf() !== -1`
- `for...of` → `for...in` + `hasOwnProperty`

#### 2. 前后端匹配方案
**设备身份识别**: 基于ESP32芯片MAC生成唯一ID
```cpp
void generateDeviceID() {
    uint64_t chipid = ESP.getEfuseMac();
    snprintf(device_id, sizeof(device_id), "MC-%04X%08X", 
             (uint32_t)(chipid >> 32), (uint32_t)chipid);
}
```

**数据流**:
```
设备(ESP32) → HTTP POST → 云API → 数据库 → Web应用
```

**设备首次启动流程**:
1. 生成/读取设备ID → Flash存储
2. 尝试连接WiFi → 无配置则启动AP配网
3. 向服务器注册 → POST /devices/register
4. 绑定到用户 → 扫码/输入设备ID
5. 开始正常工作

#### 3. 多设备数据隔离方案
**场景**: 4个设备 (2个自家 + 2个送朋友)

**避免类似问题的建议**:
1. 分批次提交，避免 `git add -A` 包含大目录
2. 检查 `.gitignore`，确保 `venv/`、`node_modules/` 被忽略
3. 长操作分多个回合进行，防止单一会话上下文过大

## 【系统知识】会话卡住诊断方法

**检查清单**:
- [ ] `sessions_list` 查看会话 token 使用量
- [ ] `sessions_history` 查看最后事件
- [ ] `sessions_send` 测试是否能正常发送消息

**常见原因**:
1. 上下文过大触发 compaction 后状态异常
2. 会话文件损坏
3. Gateway 与文件系统状态不一致

**修复步骤**:
1. 删除异常会话文件 (`rm .../sessions/<session_id>.jsonl`)
2. 让用户重新发送消息触发重建
3. 如无效，考虑重启 Gateway

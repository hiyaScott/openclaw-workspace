# 游戏音频设计速查表

## 交互音乐技术对比

| 技术 | 核心思想 | 复杂度 | 适用场景 |
|------|----------|--------|----------|
| **Horizontal Resequencing** | 水平切换音乐片段 | 中 | 区域切换、剧情分支 |
| **Vertical Remixing** | 垂直叠加/移除层 | 低 | 强度变化、战斗升级 |
| **Stingers** | 短促触发音 | 低 | 事件标记、反馈 |
| **Transitions** | 过渡片段 | 中 | 平滑切换 |

## 垂直混音层结构

```
Layer 3 (顶层): 特效/点缀
Layer 2 (中层): 旋律/配器  
Layer 1 (基础层): 和声/节奏
```

- Fade Time: 0.5-3 秒
- 基础层始终播放
- 层间可非同步进入

## 无缝循环检查清单

- [ ] 零交叉点切割
- [ ] 波形方向一致
- [ ] 预留混响尾音
- [ ] 循环长度 ≥30 秒
- [ ] 乐句结构完整

## 音效随机化参数

| 参数 | 建议范围 | 效果 |
|------|----------|------|
| 音量随机 | ±3-6dB | 自然感 |
| 音高随机 | ±1-2 半音 | 变化感 |
| 采样数量 | 3-5 个 | 消除重复 |

## 3D vs 2D 音频选择

**使用 3D**：
- 环境音效
- 空间定位声源
- NPC/敌人音效

**使用 2D**：
- 背景音乐
- UI 音效
- 玩家自身音效

## Web Audio API 核心节点

```javascript
AudioContext        // 音频上下文
OscillatorNode      // 振荡器（生成声音）
GainNode            // 音量控制
BiquadFilterNode    // 滤波器
ConvolverNode       // 混响
AnalyserNode        // 频谱分析
```

## 音频格式建议

| 类型 | 格式 | 参数 |
|------|------|------|
| 短音效 | WAV | 单声道 16bit 44.1kHz |
| 音乐 | OGG | 128-192kbps |
| 长音频 | OGG | 流式播放 |

## 音乐游戏反馈层级

1. **即时反馈**：击中/未击中音效
2. **进度反馈**：Combo、分数
3. **状态反馈**：血条、能量
4. **里程碑反馈**：关卡完成、高分

## 节奏同步要点

- MIDI/MOD：可实时变速
- 预渲染：多速度版本或时间拉伸
- 在乐句边界处切换
- 使用 look-ahead 调度器

## 延迟控制

- Web Audio API 延迟：通常 10-50ms
- 使用 `currentTime + offset` 精确调度
- 考虑显示延迟（通常 16ms@60fps）
